Caffe模型转PaddlePaddle的Fluid版本预测模型

2020-03-23
   
PaddlePaddle 人工智能 夜雨飘零 深度学习
    0 0

前言

有不少开发者在学习深度学习框架的时候会开源一些训练好的模型，我们可以使用这些模型来运用到我们自己的项目中。如果使用的是同一个深度学习框架，那就很方便，可以直接使用，但是如果时不同深度学习框架，我们就要对模型转换一下。下面我们就介绍如何把Caffe的模型转换成PaddlePaddle的Fluid模型。

环境准备

  • 在线安装最新发布的PaddlePaddle，可以使用pip命令直接在线安装PaddlePaddle。
1pip install paddlepaddle
  • 下在安装最新的PaddlePaddle，可以在下面选择适合自己PaddlePaddle的版本，这里下载的是最新编译得到的，然后使用pip命令安装。
1http://www.paddlepaddle.org/documentation/docs/zh/0.14.0/new_docs/beginners_guide/install/install_doc.html#id26
  • 克隆PaddlePaddle下的models源码。
1git clone https://github.com/PaddlePaddle/models.git

转换模型

  • 进入到上一步克隆的models的caffe2fluid目录。
1cd models/fluid/image_classification/caffe2fluid/
  • 下载转换时所需要的依赖的Python文件。
1cd proto/ && wget https://raw.githubusercontent.com/ethereon/caffe-tensorflow/master/kaffe/caffe/caffepb.py

修改刚下载的文件名：

1mv caffepb.py caffe_pb2.py
  • 获取需要转换的Caffe模型，笔者是参考以下这个开源获取的：
1https://gist.github.com/ksimonyan/211839e770f7b538e2d8

首先是获取VGG_ILSVRC_16_layers_deploy.prototxt网络文件。

1cd ../ && wget https://gist.githubusercontent.com/ksimonyan/211839e770f7b538e2d8/raw/ded9363bd93ec0c770134f4e387d8aaaaa2407ce/VGG_ILSVRC_16_layers_deploy.prototxt

其次是获取VGG_ILSVRC_16_layers.caffemodel权重文件。

1wget http://www.robots.ox.ac.uk/~vgg/software/very_deep/caffe/VGG_ILSVRC_16_layers.caffemodel
  • 把Caffe模型转换成Fluid版本的网络结构文件和权重文件，其中VGG16.py是PaddlePaddle定义网络结构的Python文件，VGG16.npy是网络的权重文件。
1python convert.py VGG_ILSVRC_16_layers_deploy.prototxt \
2        --caffemodel VGG_ILSVRC_16_layers.caffemodel \
3        --data-output-path VGG16.npy \
4        --code-output-path VGG16.py

  • 使用PaddlePaddle的网络结构文件和权重文件生成预测模型文件。
1python VGG16.py VGG16.npy ./fluid_models
  • 执行上面的命令之后，就可以生成预测模型了，并存放在当前目录的fluid_models目录下，一共有两文件，分别是model和params，这个跟我们使用paddle.fluid.io.save_inference_model接口是一样的，接口文档在这里。在下一步我们会使用这个模型文件来预测我们的图片。

测试预测模型

获得预测模型之后，我们可以使用它来在PaddlePaddle预测图像，首先要编写一个PaddlePaddle的预测程序：

 1# coding=utf-8
 2import os
 3import time
 4from PIL import Image
 5import numpy as np
 6import paddle.v2 as paddle
 7import paddle.fluid as fluid
 8
 9
10def load_image(file):
11    im = Image.open(file)
12    im = im.resize((224, 224), Image.ANTIALIAS)
13    im = np.array(im).astype(np.float32)
14    # PIL打开图片存储顺序为H(高度)，W(宽度)，C(通道)。
15    # PaddlePaddle要求数据顺序为CHW，所以需要转换顺序。
16    im = im.transpose((2, 0, 1))
17    # CIFAR训练图片通道顺序为B(蓝),G(绿),R(红),
18    # 而PIL打开图片默认通道顺序为RGB,因为需要交换通道。
19    im = im[(2, 1, 0), :, :]  # BGR
20    # 减去均值
21    mean = [123.68, 116.78, 103.94]
22    mean = np.array(mean, dtype=np.float32)
23    mean = mean[:, np.newaxis, np.newaxis]
24    im -= mean
25
26    return im
27
28
29def infer_one(image_path, use_cuda, model_path, model_filename, params_filename):
30    # 是否使用GPU
31    place = fluid.CUDAPlace(0) if use_cuda else fluid.CPUPlace()
32    # 生成调试器
33    exe = fluid.Executor(place)
34
35    inference_scope = fluid.core.Scope()
36    with fluid.scope_guard(inference_scope):
37        # 加载模型
38        [inference_program, feed_target_names, fetch_targets] = fluid.io.load_inference_model(model_path, exe, model_filename=model_filename, params_filename=params_filename)
39
40        # 获取预测数据
41        test_datas = [load_image(image_path)]
42        test_data = np.array(test_datas)
43
44        # 开始预测
45        results = exe.run(inference_program,
46                          feed={feed_target_names[0]: test_data},
47                          fetch_list=fetch_targets)
48
49        results = np.argsort(-results[0])
50
51        result = results[0][0]
52
53        print("infer label is: %d" % result)
54
55
56if __name__ == '__main__':
57    image_path = "0b77aba2-9557-11e8-a47a-c8ff285a4317.jpg"
58    use_cuda = False
59    model_path = "fluid_models/"
60    model_filename = "model"
61    params_filename = "params"
62    infer_one(image_path, use_cuda, model_path, model_filename, params_filename)

使用上面的程序就是使用转换的模型来预测图片了。要注意训练模型时对图片的处理。
