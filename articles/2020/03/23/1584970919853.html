*本篇文章基于 PaddlePaddle 0.10.0、Python 2.7

前言

在上一篇文章中介绍了验证码的识别，但是使用的传统的验证码分割，然后通过图像分类的方法来实现验证码的识别的，这中方法比较繁琐，工作量比较多。在本篇文章会介绍验证码端到端的识别，直接一步到位，不用图像分割那么麻烦了。好吧，现在开始吧！

数据集介绍

在本篇文章中同样是使用方正系统的验证码，该数据集在上一篇文章《我的PaddlePaddle学习之路》笔记五——验证码的识别已有介绍，在这里我就不介绍了，需要了解的可以点击链接去到上一篇文章查看。

获取验证码

下载验证码和修改验证码同样在上一篇文章有介绍，如果读者需要同样可以回到上一篇文章查看。
验证码我们有了，有看过上一篇文章的读者会第一反应说还缺图像列表。没错，训练和测试都需要一个图像列表

把图像转成灰度图

在生成列表之前，我们还有对图像做一些处理，就是把图像灰度化。
**注意：**在此之前应该把图像文件命名，文件名为验证码对应的字符，并把所有的验证码放在data_temp
然后执行以下的程序批量处理

 1# coding=utf-8
 2import os
 3from PIL import Image
 4
 5def Image2GRAY(path):
 6    # 获取临时文件夹中的所有图像路径
 7    imgs = os.listdir(path)
 8    i = 0
 9    for img in imgs:
10        # 每10个数据取一个作为测试数据，剩下的作为训练数据
11        if i % 10 == 0:
12            # 使图像灰度化并保存
13            im = Image.open(path + '/' + img).convert('L')
14            im.save('data/test_data/' + img)
15        else:
16            # 使图像灰度化并保存
17            im = Image.open(path + '/' + img).convert('L')
18            im.save('data/train_data/' + img)
19        i = i + 1
20
21if __name__ == '__main__':
22    # 临时数据存放路径
23    path = 'data/data_temp'
24    Image2GRAY(path)

生成图像列表

经过上面一步，在data/train_data我们有了训练数据集，data/test_data测试数据集。然后就在这两个文件夹下生成对应的图像列表。
首先我们要了解图像列表的格式要求，我们来看看它的格式是怎样的

110iw.png	10iw
2218j.png	218j
328hi.png	28hi
43n1g.png	3n1g
547q7.png	47q7
64ju5.png	4ju5
74uqh.png	4uqh

这个图像类别是以Tab键区分路径和label的，了解图像列表的格式要求之后，那么我们就编写一个程序来生成这样格式的一个图像列表。代码如下：

 1# coding=utf-8
 2import os
 3
 4class CreateDataList:
 5    def __init__(self):
 6        pass
 7
 8    def createDataList(self, data_path, isTrain):
 9        # 判断生成的列表是训练图像列表还是测试图像列表
10        if isTrain:
11            list_name = 'trainer.list'
12        else:
13            list_name = 'test.list'
14        list_path = os.path.join(data_path, list_name)
15        # 判断该列表是否存在，如果存在就删除，避免在生成图像列表时把该路径也写进去了
16        if os.path.exists(list_path):
17            os.remove(list_path)
18        # 读取所有的图像路径，此时图像列表不存在，就不用担心写入非图像文件路径了
19        imgs = os.listdir(data_path)
20        for img in imgs:
21            name = img.split('.')[0]
22            with open(list_path, 'a') as f:
23                # 写入图像路径和label，用Tab隔开
24                f.write(img + '\t' + name + '\n')
25
26if __name__ == '__main__':
27    createDataList = CreateDataList()
28    # 生成训练图像列表
29    createDataList.createDataList('data/train_data/', True)
30    # 生成测试图像列表
31    createDataList.createDataList('data/test_data/', False)

经过上面的程序，会在data/train_data生成图像列表trainer.list，会在data/test_data生成图像列表test.list。到这里，我们的数据集已经准备好了，准备开始使用数据集训练了。

数据的读取

读取数据成list

数据列表是有了，但是我们使用它就要用到文件读取，生成一个我们方便使用的的数据格式。在本例子项目中，我把图像的路径和label生成是一个list。读取方式如下：

 1def get_file_list(image_file_list):
 2    '''
 3    生成用于训练和测试数据的文件列表。
 4    :param image_file_list: 图像文件和列表文件的路径
 5    :type image_file_list: str
 6    '''
 7    dirname = os.path.dirname(image_file_list)
 8    path_list = []
 9    with open(image_file_list) as f:
10        for line in f:
11            # 使用Tab键分离路径和label
12            line_split = line.strip().split('\t')
13            filename = line_split[0].strip()
14            path = os.path.join(dirname, filename)
15            label = line_split[1].strip()
16            if label:
17                path_list.append((path, label))
18
19    return path_list

生成和读取标签字典

在这个项目中，要使用到我们之前没有使用过的文件：标签字典。这个标签字典是训练数据集中出现的字符，如：

1r	81
24	77
3h	75
4i	74
52	72

通过每个字符的key就可以找到对应的字符了。
我们要编写一个从训练数据集的list中获取所有的字符，并生成一个标签字典

 1def build_label_dict(file_list, save_path):
 2    """
 3    从训练数据建立标签字典
 4    :param file_list: 包含标签的训练数据列表
 5    :type file_list: list
 6    :params save_path: 保存标签字典的路径
 7    :type save_path: str
 8    """
 9    values = defaultdict(int)
10    for path, label in file_list:
11        for c in label:
12            if c:
13                values[c] += 1
14
15    values['<unk>'] = 0
16    with open(save_path, "w") as f:
17        for v, count in sorted(
18                values.iteritems(), key=lambda x: x[1], reverse=True):
19            f.write("%s\t%d\n" % (v, count))

然后只要传入在上一步读取到的train_file_list和保存标签字典的路径就可以生成标签字典了。

读取训练和测试的数据

如果学习前面几个例子的，应该会知道trainer传入的数据是reader的，在上面获取的训练数据和测试数据都是list类型的，我们要把它转成reader类型的。同下面的程序，把训练和测试的数据根据其路径来加载成一维向量

 1# coding=utf-8
 2import cv2
 3import paddle.v2 as paddle
 4
 5class Reader(object):
 6    def __init__(self, char_dict, image_shape):
 7        '''
 8        :param char_dict: 标签的字典类
 9        :type char_dict: class
10        :param image_shape: 图像的固定形状
11        :type image_shape: tuple
12        '''
13        self.image_shape = image_shape
14        self.char_dict = char_dict
15
16    def train_reader(self, file_list):
17        '''
18        训练读取数据
19        :param file_list: 用预训练的图像列表，包含标签和图像路径
20        :type file_list: list
21        '''
22        def reader():
23            UNK_ID = self.char_dict['<unk>']
24            for image_path, label in file_list:
25                label = [self.char_dict.get(c, UNK_ID) for c in label]
26                yield self.load_image(image_path), label
27        return reader
28
29    def load_image(self, path):
30        '''
31        加载图像并将其转换为1维矢量
32        :param path: 图像数据的路径
33        :type path: str
34        '''
35        image = paddle.image.load_image(path,is_color=False)
36        # 将所有图像调整为固定形状
37        if self.image_shape:
38            image = cv2.resize(
39                image, self.image_shape, interpolation=cv2.INTER_CUBIC)
40        image = image.flatten() / 255.
41        return image

我们通过传入标签字典和图像的大小(宽度,高度)获取reader

1my_reader = Reader(char_dict=char_dict, image_shape=IMAGE_SHAPE)

然后通过执行下面的方法，同时传入训练的list：train_file_list和测试的list：test_file_list就可以生成reader了。

定义网络模型

这次使用的网络模型不是单纯的CNN模型了，还有结合了RNN来映射字符的分布和使用CTC来计算CTC任务的成本，具体是如何定义的呢，请往下细看。
跟之前一样，我们同样要定义数据的和label，更之前不一样的是这次我们定义数据的时候指定了宽度和高度，因为我们这个数据集只长方形的。
在定义label的时候，之前我们要传入类别的总数，我们这次还是同样的道理。还记得上一步获得的标签字典吧，标签字典就是我们训练集的所有出现过字符，只要获取字符的大小就可以了。

1# 获取字典大小
2dict_size = len(char_dict)

以下就是类初始化的数据和定义数据和label的操作：

 1class Model(object):
 2    def __init__(self, num_classes, shape, is_infer=False):
 3        '''
 4        :param num_classes: 字符字典的大小
 5        :type num_classes: int
 6        :param shape: 输入图像的大小
 7        :type shape: tuple of 2 int
 8        :param is_infer: 是否用于预测
 9        :type shape: bool
10        '''
11        self.num_classes = num_classes
12        self.shape = shape
13        self.is_infer = is_infer
14        self.image_vector_size = shape[0] * shape[1]
15
16        self.__declare_input_layers__()
17        self.__build_nn__()
18
19    def __declare_input_layers__(self):
20        '''
21        定义输入层
22        '''
23        # 图像输入为一个浮动向量
24        self.image = paddle.layer.data(
25            name='image',
26            type=paddle.data_type.dense_vector(self.image_vector_size),
27            # shape是(宽度,高度)
28            height=self.shape[1],
29            width=self.shape[0])
30
31        # 将标签输入为ID列表
32        if not self.is_infer:
33            self.label = paddle.layer.data(
34                name='label',
35                type=paddle.data_type.integer_value_sequence(self.num_classes))

定义网络模型，该网络模型
首先是通过CNN获取图像的特征，
然后使用这些特征来输出展开成一系列特征向量，
然后使用RNN向前和向后捕获序列信息，
然后将RNN的输出映射到字符分布，
最后使用扭曲CTC来计算CTC任务的成本，获得了cost和额外层。

 1def __build_nn__(self):
 2    '''
 3    建立网络拓扑
 4    '''
 5    # 通过CNN获取图像特征
 6    def conv_block(ipt, num_filter, groups, num_channels=None):
 7        return paddle.networks.img_conv_group(
 8            input=ipt,
 9            num_channels=num_channels,
10            conv_padding=1,
11            conv_num_filter=[num_filter] * groups,
12            conv_filter_size=3,
13            conv_act=paddle.activation.Relu(),
14            conv_with_batchnorm=True,
15            pool_size=2,
16            pool_stride=2, )
17
18    # 因为是灰度图所以最后一个参数是1
19    conv1 = conv_block(self.image, 16, 2, 1)
20    conv2 = conv_block(conv1, 32, 2)
21    conv3 = conv_block(conv2, 64, 2)
22    conv_features = conv_block(conv3, 128, 2)
23
24    # 将CNN的输出展开成一系列特征向量。
25    sliced_feature = paddle.layer.block_expand(
26        input=conv_features,
27        num_channels=128,
28        stride_x=1,
29        stride_y=1,
30        block_x=1,
31        block_y=11)
32
33    # 使用RNN向前和向后捕获序列信息。
34    gru_forward = paddle.networks.simple_gru(
35        input=sliced_feature, size=128, act=paddle.activation.Relu())
36    gru_backward = paddle.networks.simple_gru(
37        input=sliced_feature,
38        size=128,
39        act=paddle.activation.Relu(),
40        reverse=True)
41
42    # 将RNN的输出映射到字符分布。
43    self.output = paddle.layer.fc(input=[gru_forward, gru_backward],
44                                  size=self.num_classes + 1,
45                                  act=paddle.activation.Linear())
46
47    self.log_probs = paddle.layer.mixed(
48        input=paddle.layer.identity_projection(input=self.output),
49        act=paddle.activation.Softmax())
50
51    # 使用扭曲CTC来计算CTC任务的成本。
52    if not self.is_infer:
53        # 定义cost
54        self.cost = paddle.layer.warp_ctc(
55            input=self.output,
56            label=self.label,
57            size=self.num_classes + 1,
58            norm_by_times=True,
59            blank=self.num_classes)
60        # 定义额外层
61        self.eval = paddle.evaluator.ctc_error(input=self.output, label=self.label)

最后通过调用该类就可以获取到模型了，传入的参数是
dict_size是标签字典的大小，在上面有介绍是用来生成label的
IMAGE_SHAPE这个是图像的宽度和高度，格式是：(宽度,高度)

训练事件event_handler，通过这个训练事件我们可以在训练的时候处理一下事情，如输出训练日志用于观察训练的效果，方便分析模型的性能。还可以保持模型，用于之后可预测或者再训练。定义如下：

 1# 训练事件
 2def event_handler(event):
 3    if isinstance(event, paddle.event.EndIteration):
 4        if event.batch_id % 100 == 0:
 5            print("Pass %d, batch %d, Samples %d, Cost %f, Eval %s" %
 6                  (event.pass_id, event.batch_id, event.batch_id *
 7                   BATCH_SIZE, event.cost, event.metrics))
 8
 9    if isinstance(event, paddle.event.EndPass):
10        # 这里由于训练和测试数据共享相同的格式
11        # 我们仍然使用reader.train_reader来读取测试数据
12        test_reader = paddle.batch(
13            my_reader.train_reader(test_file_list),
14            batch_size=BATCH_SIZE)
15        result = trainer.test(reader=test_reader, feeding=feeding)
16        print("Test %d, Cost %f, Eval %s" % (event.pass_id, result.cost, result.metrics))
17        # 检查保存model的路径是否存在，如果不存在就创建
18        if not os.path.exists(model_save_dir):
19            os.mkdir(model_save_dir)
20        with gzip.open(
21                os.path.join(model_save_dir, "params_pass.tar.gz"), "w") as f:
22            trainer.save_parameter_to_tar(f)

最后的num_passes就是训练轮数。

11    paddle.init(use_gpu=True, trainer_count=1)

13    parameters = paddle.parameters.Parameters.from_tar(gzip.open(model_path))
14    # 获取网络模型
15    model = Model(dict_size, image_shape, is_infer=True)
16    # 获取预测器
17    inferer = paddle.inference.Inference(output_layer=model.log_probs, parameters=parameters)
18    # 加载数据
19    test_batch = [[my_reader.load_image(img_path)]]
20    # 开始预测
21    return start_infer(inferer, test_batch, reversed_char_dict)

上面使用的反转的标签字典定义如下，通过标签字典的文件即可生成反转的标签字典

1def load_reverse_dict(dict_path):
2    """
3    从字典路径加载反转的标签字典
4    :param dict_path: 标签字典的路径
5    :type dict_path: str
6    """
7    return dict((idx, line.strip().split("\t")[0])

通过传入上面获取是的inferer和图像的一维向量，还有反转的标签字典就可以进行预测了。

 1def start_infer(inferer, test_batch, reversed_char_dict):
 2    # 获取初步预测结果
 3    infer_results = inferer.infer(input=test_batch)
 4    num_steps = len(infer_results) // len(test_batch)
 5    probs_split = [
 6        infer_results[i * num_steps:(i + 1) * num_steps]
 7        for i in range(0, len(test_batch))]
 8    # 最佳路径解码
 9    result = ''
10    for i, probs in enumerate(probs_split):
11        result = ctc_greedy_decoder(
12            probs_seq=probs, vocabulary=reversed_char_dict)
13    return result

这个还使用到了最佳路径解码，使用的解码器如下：

 1def ctc_greedy_decoder(probs_seq, vocabulary):
 2    """CTC贪婪（最佳路径）解码器。
 3    由最可能的令牌组成的路径被进一步后处理
 4    删除连续的重复和所有的空白。
 5    :param probs_seq: 每个词汇表上概率的二维列表字符。
 6                      每个元素都是浮点概率列表为一个字符。
 7    :type probs_seq: list
 8    :param vocabulary: 词汇表
 9    :type vocabulary: list
10    :return: 解码结果字符串
11    :rtype: baseline
12    """
13    # 尺寸验证
14    for probs in probs_seq:
15        if not len(probs) == len(vocabulary) + 1:
16            raise ValueError("probs_seq dimension mismatchedd with vocabulary")
17    # argmax以获得每个时间步长的最佳指标
18    max_index_list = list(np.array(probs_seq).argmax(axis=1))
19    # 删除连续的重复索引
20    index_list = [index_group[0] for index_group in groupby(max_index_list)]
21    # 删除空白索引
22    blank_index = len(vocabulary)
23    index_list = [index for index in index_list if index != blank_index]
24    # 将索引列表转换为字符串
25    return ''.join([vocabulary[index] for index in index_list])
