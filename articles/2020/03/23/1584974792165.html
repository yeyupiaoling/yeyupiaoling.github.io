前言

本系列教程中，前面介绍的都没有保存模型，训练之后也就结束了。那么本章就介绍如果在训练过程中保存模型，用于之后预测或者恢复训练，又或者由于其他数据集的预训练模型。本章会介绍三种保存模型和使用模型的方式。

训练模型

在训练模型的过程中我们可以随时保存模型，当时也可以在训练开始之前加载之前训练过程的模型。为了介绍这三个保存模型的方式，一共编写了三个Python程序进行介绍，分别是save_infer_model.py、 save_use_params_model.py、save_use_persistables_model.py。

导入相关的依赖库

1import os
2import shutil
3import paddle as paddle
4import paddle.dataset.cifar as cifar
5import paddle.fluid as fluid

定义一个残差神经网络，这个是目前比较常用的一个网络。该神经模型可以通过增加网络的深度达到提高识别率，而不会像其他过去的神经模型那样，当网络继续加深时,反而会损失精度。

 1# 定义残差神经网络（ResNet）
 2def resnet_cifar10(ipt, class_dim):
 3    def conv_bn_layer(input,
 4                      ch_out,
 5                      filter_size,
 6                      stride,
 7                      padding,
 8                      act='relu',
 9                      bias_attr=False):
10        tmp = fluid.layers.conv2d(
11            input=input,
12            filter_size=filter_size,
13            num_filters=ch_out,
14            stride=stride,
15            padding=padding,
16            bias_attr=bias_attr)
17        return fluid.layers.batch_norm(input=tmp, act=act)
18
19    def shortcut(input, ch_in, ch_out, stride):
20        if ch_in != ch_out:
21            return conv_bn_layer(input, ch_out, 1, stride, 0, None)
22        else:
23            return input
24
25    def basicblock(input, ch_in, ch_out, stride):
26        tmp = conv_bn_layer(input, ch_out, 3, stride, 1)
27        tmp = conv_bn_layer(tmp, ch_out, 3, 1, 1, act=None, bias_attr=True)
28        short = shortcut(input, ch_in, ch_out, stride)
29        return fluid.layers.elementwise_add(x=tmp, y=short, act='relu')
30
31    # 残差块
32    def layer_warp(block_func, input, ch_in, ch_out, count, stride):
33        tmp = block_func(input, ch_in, ch_out, stride)
34        for i in range(1, count):
35            tmp = block_func(tmp, ch_out, ch_out, 1)
36        return tmp
37
38    conv1 = conv_bn_layer(ipt, ch_out=16, filter_size=3, stride=1, padding=1)
39    res1 = layer_warp(basicblock, conv1, 16, 16, 5, 1)
40    res2 = layer_warp(basicblock, res1, 16, 32, 5, 2)
41    res3 = layer_warp(basicblock, res2, 32, 64, 5, 2)
42    pool = fluid.layers.pool2d(input=res3, pool_size=8, pool_type='avg', pool_stride=1)
43    predict = fluid.layers.fc(input=pool, size=class_dim, act='softmax')
44    return predict

定义输出成，这里使用的数据集是cifar数据集，这个数据集的图片是宽高都为32的3通道图片，所以这里定义的图片输入层的shape是[3, 32, 32]。

1# 定义输入层
2image = fluid.layers.data(name='image', shape=[3, 32, 32], dtype='float32')
3label = fluid.layers.data(name='label', shape=[1], dtype='int64')

获取残差神经网络的分类器，并指定分类大小是10，因为这个数据集有10个类别。

1# 获取分类器
2model = resnet_cifar10(image, 10)

获取交叉熵损失函数和平均准确率，模型获取的准确率是Top1的。

1# 获取损失函数和准确率函数
2cost = fluid.layers.cross_entropy(input=model, label=label)
3avg_cost = fluid.layers.mean(cost)
4acc = fluid.layers.accuracy(input=model, label=label)

获取测试程序，用于之后的测试使。

1# 获取训练和测试程序
2test_program = fluid.default_main_program().clone(for_test=True)

定义优化方法。

1# 定义优化方法
2optimizer = fluid.optimizer.AdamOptimizer(learning_rate=1e-3)
3opts = optimizer.minimize(avg_cost)

获取训练和测试数据，使用的是cifar数据集，cifar数据集有两种，一种是100个类别的，一种是10个类别的，这里使用的是10个类别的。

1# 获取CIFART数据
2train_reader = paddle.batch(cifar.train10(), batch_size=32)
3test_reader = paddle.batch(cifar.test10(), batch_size=32)

创建执行器，因为我们使用的网络是一个比较大的网络，而且图片也比之前的灰度图要大很多。之前的MNIST数据集的每张图片大小784，而现在的是3072。当然主要是网络比之前的要大很多很多，如果使用CPU训练，速度是非常慢的，所以最好使用GPU进行训练。

1# 创建执行器，最好使用GPU，CPU速度太慢了
2# place = fluid.CPUPlace()
3place = fluid.CUDAPlace(0)
4exe = fluid.Executor(place)
5# 进行参数初始化
6exe.run(fluid.default_startup_program())

加载模型

创建执行器之后，就可以加载之前训练的模型了，有两种加载模型的方式，对应着两种保存模型的方式。这两种模型，可以只使用一种就可以。

  • save_use_params_model.py加载之前训练保存的参数模型，对应的保存接口是fluid.io.save_params。使用这些模型参数初始化网络参数，进行训练
1# 加载之前训练过的参数模型
2save_path = 'models/params_model/'
3if os.path.exists(save_path):
4    print('使用参数模型作为预训练模型')
5    fluid.io.load_params(executor=exe, dirname=save_path)
  • save_use_persistables_model.py加载之前训练保存的持久化变量模型，对应的保存接口是fluid.io.save_persistables。使用这些模型参数初始化网络参数，进行训练。
1# 加载之前训练过的检查点模型
2save_path = 'models/persistables_model/'
3if os.path.exists(save_path):
4    print('使用持久化变量模型作为预训练模型')
5    fluid.io.load_persistables(executor=exe, dirname=save_path)

开始训练模型。

 1# 定义输入数据维度
 2feeder = fluid.DataFeeder(place=place, feed_list=[image, label])

 4for pass_id in range(10):

 7        train_cost, train_acc = exe.run(program=fluid.default_main_program(),
 8                                        feed=feeder.feed(data),

10        # 每100个batch打印一次信息
11        if batch_id % 100 == 0:
12            print('Pass:%d, Batch:%d, Cost:%0.5f, Accuracy:%0.5f' %
13                  (pass_id, batch_id, train_cost[0], train_acc[0]))
14
15    # 进行测试
16    test_accs = []
17    test_costs = []
18    for batch_id, data in enumerate(test_reader()):
19        test_cost, test_acc = exe.run(program=test_program,
20                                      feed=feeder.feed(data),
21                                      fetch_list=[avg_cost, acc])
22        test_accs.append(test_acc[0])
23        test_costs.append(test_cost[0])
24    # 求测试结果的平均值
25    test_cost = (sum(test_costs) / len(test_costs))
26    test_acc = (sum(test_accs) / len(test_accs))
27    print('Test:%d, Cost:%0.5f, Accuracy:%0.5f' % (pass_id, test_cost, test_acc))

保存模型

训练结束之后，就可以进行保存模型。当然也不一样要全部训练结束才保存模型，我们可以在每一个Pass训练结束之后保存一次模型。这里使用三个程序分别保存，当然也可以一次全部保存。

  • save_infer_model.py保存预测模型，之后用于预测图像。通过使用这个方式保存的模型，之后预测是非常方便的，具体可以阅读预测部分。
1# 保存预测模型
2save_path = 'models/infer_model/'
3# 删除旧的模型文件
4shutil.rmtree(save_path, ignore_errors=True)
5# 创建保持模型文件目录
6os.makedirs(save_path)
7# 保存预测模型
8fluid.io.save_inference_model(save_path, feeded_var_names=[image.name], target_vars=[model], executor=exe)
  • save_use_params_model.py保存参数模型，之后用于初始化模型，进行训练。
1# 保存参数模型
2save_path = 'models/params_model/'
3# 删除旧的模型文件
4shutil.rmtree(save_path, ignore_errors=True)
5# 创建保持模型文件目录
6os.makedirs(save_path)
7# 保存参数模型
8fluid.io.save_params(executor=exe, dirname=save_path)
  • save_use_persistables_model.py保存持久化变量模型，之后用于初始化模型，进行训练。
1# 保存持久化变量模型
2save_path = 'models/persistables_model/'
3# 删除旧的模型文件
4shutil.rmtree(save_path, ignore_errors=True)
5# 创建保持模型文件目录
6os.makedirs(save_path)
7# 保存持久化变量模型
8fluid.io.save_persistables(executor=exe, dirname=save_path)

预测

在训练的时候使用fluid.io.save_inference_model接口保存的模型，可以通过以下use_infer_model.py程序预测，通过这个程序，读者会发现通过这个接口保存的模型，再次预测是非常简单的。

导入相关的依赖库

1import paddle.fluid as fluid
2from PIL import Image
3import numpy as np

创建一个执行器，预测图片可以使用CPU执行，这个速度不会太慢。

1# 创建执行器
2place = fluid.CPUPlace()
3exe = fluid.Executor(place)
4exe.run(fluid.default_startup_program())

加载模型，这个是整个预测程序的重点，通过加载预测模型我们就可以轻松获取得到一个预测程序，输出参数的名称，以及分类器的输出。

1# 保存预测模型路径
2save_path = 'models/infer_model/'
3# 从模型中获取预测程序、输入数据名称列表、分类器
4[infer_program, feeded_var_names, target_var] = fluid.io.load_inference_model(dirname=save_path, executor=exe)

定义一个图像预处理的函数，这个函数可以统一图像大小，修改图像的存储顺序和图片的通道顺序，转换成numpy数据。

 1# 预处理图片
 2def load_image(file):
 3    im = Image.open(file)
 4    im = im.resize((32, 32), Image.ANTIALIAS)
 5    im = np.array(im).astype(np.float32)
 6    # PIL打开图片存储顺序为H(高度)，W(宽度)，C(通道)。
 7    # PaddlePaddle要求数据顺序为CHW，所以需要转换顺序。
 8    im = im.transpose((2, 0, 1))
 9    # CIFAR训练图片通道顺序为B(蓝),G(绿),R(红),
10    # 而PIL打开图片默认通道顺序为RGB,因为需要交换通道。
11    im = im[(2, 1, 0), :, :]  # BGR
12    im = im / 255.0
13    im = np.expand_dims(im, axis=0)
14    return im

获取数据并进行预测。这里对比之前的预测方式，不需要再输入一个模拟的标签，因为在保存模型的时候，已经对这部分进行修剪，去掉了这部分不必要的输入。

1# 获取图片数据
2img = load_image('image/cat.png')
3
4# 执行预测
5result = exe.run(program=infer_program,
6                 feed={feeded_var_names[0]: img},
7                 fetch_list=target_var)

执行预测之后，得到一个数组，这个数组是表示每个类别的概率，获取最大概率的标签，并根据标签获取获取该类的名称。

1# 显示图片并输出结果最大的label
2lab = np.argsort(result)[0][0][-1]
3
4names = ['飞机', '汽车', '鸟', '猫', '鹿', '狗', '青蛙', '马', '船', '卡车']
5
6print('预测结果标签为：%d， 名称为：%s， 概率为：%f' % (lab, names[lab], result[0][0][lab]))
