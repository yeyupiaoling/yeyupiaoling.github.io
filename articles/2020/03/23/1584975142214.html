前言

我们在第六章介绍了生成对抗网络，并使用生成对抗网络训练mnist数据集，生成手写数字图片。那么本章我们将使用对抗生成网络训练我们自己的图片数据集，并生成图片。在第六章中我们使用的黑白的单通道图片，在这一章中，我们使用的是3通道的彩色图。

GitHub地址：https://github.com/yeyupiaoling/LearnPaddle2/tree/master/note13

定义数据读取

我们首先创建一个image_reader.py文件，用于读取我们自己定义的图片数据集。首先导入所需的依赖包。

1import os
2import random
3from multiprocessing import cpu_count
4import numpy as np
5import paddle
6from PIL import Image

这里的图片预处理主要是对图片进行等比例压缩和中心裁剪，这里为了避免图片在图片在resize时出现变形的情况，导致训练生成的图片不是我们真实图片的样子。这里为了增强数据集，做了随机水平翻转。最后在处理图片的时候，为了避免数据集中有单通道图片导致训练中断，所以还把单通道图转成3通道图片。

 1# 测试图片的预处理
 2def train_mapper(sample):
 3    img, crop_size = sample
 4    img = Image.open(img)
 5    # 随机水平翻转
 6    r1 = random.random()
 7    if r1 > 0.5:
 8        img = img.transpose(Image.FLIP_LEFT_RIGHT)
 9    # 等比例缩放和中心裁剪
10    width = img.size[0]
11    height = img.size[1]
12    if width < height:
13        ratio = width / crop_size
14        width = width / ratio
15        height = height / ratio
16        img = img.resize((int(width), int(height)), Image.ANTIALIAS)
17        height = height / 2
18        crop_size2 = crop_size / 2
19        box = (0, int(height - crop_size2), int(width), int(height + crop_size2))
20    else:
21        ratio = height / crop_size
22        height = height / ratio
23        width = width / ratio
24        img = img.resize((int(width), int(height)), Image.ANTIALIAS)
25        width = width / 2
26        crop_size2 = crop_size / 2
27        box = (int(width - crop_size2), 0, int(width + crop_size2), int(height))
28    img = img.crop(box)
29    img = img.resize((crop_size, crop_size), Image.ANTIALIAS)
30
31    # 把单通道图变成3通道
32    if len(img.getbands()) == 1:
33        img1 = img2 = img3 = img
34        img = Image.merge('RGB', (img1, img2, img3))
35
36    # 转换成numpy值
37    img = np.array(img).astype(np.float32)
38    # 转换成CHW
39    img = img.transpose((2, 0, 1))
40    # 转换成BGR
41    img = img[(2, 1, 0), :, :] / 255.0
42    return img

在这篇文章中，我们读取数据集不需要使用到数据列表，因为我们并没有进行分类，只是把所有的图片用于训练并生成图片。所有这里只需要把文件中的所有图片都读取进行训练就 可以了。

 1# 测试的图片reader
 2def train_reader(train_image_path, crop_size):
 3    pathss = []
 4    for root, dirs, files in os.walk(train_image_path):
 5        path = [os.path.join(root, name) for name in files]
 6        pathss.extend(path)
 7
 8    def reader():
 9        for line in pathss:
10            yield line, crop_size
11
12    return paddle.reader.xmap_readers(train_mapper, reader, cpu_count(), 1024)

训练生成模型

下面创建train.py文件，用于训练对抗生成模型，并在训练过程中生成图片和保存预测模型。首先导入所需的依赖包。

1import os
2import shutil
3import numpy as np
4import paddle
5import paddle.fluid as fluid
6import matplotlib.pyplot as plt
7import image_reader

下面时定义生成器的，我们在第六章也介绍过。生成器的作用是尽可能生成满足判别器条件的图像。随着以上训练的进行，判别器不断增强自身的判别能力，而生成器也不断生成越来越逼真的图片，以欺骗判别器。生成器主要由两组全连接和BN层、两组转置卷积运算组成。唯一不同的时在生成器最后输出的大小是3，因为我们生成的图片是3通道的彩色图片，而且使用的激活函数是sigmoid，保证了输出的结果都是在0到1范围之内，这是彩色图片的颜色范围。

 1# 训练的图片大小
 2image_size = 112
 3
 4# 定义生成器
 5def Generator(y, name="G"):
 6    def deconv(x, num_filters, filter_size=5, stride=2, dilation=1, padding=2, output_size=None, act=None):
 7        return fluid.layers.conv2d_transpose(input=x,
 8                                             num_filters=num_filters,
 9                                             output_size=output_size,
10                                             filter_size=filter_size,
11                                             stride=stride,
12                                             dilation=dilation,
13                                             padding=padding,
14                                             act=act)
15
16    with fluid.unique_name.guard(name + "/"):
17        # 第一组全连接和BN层
18        y = fluid.layers.fc(y, size=2048)
19        y = fluid.layers.batch_norm(y)
20        # 第二组全连接和BN层
21        y = fluid.layers.fc(y, size=int(128 * (image_size / 4) * (image_size / 4)))
22        y = fluid.layers.batch_norm(y)
23        # 进行形状变换
24        y = fluid.layers.reshape(y, shape=[-1, 128, int((image_size / 4)), int((image_size / 4))])
25        # 第一组转置卷积运算
26        y = deconv(x=y, num_filters=128, act='relu', output_size=[int((image_size / 2)), int((image_size / 2))])
27        # 第二组转置卷积运算
28        y = deconv(x=y, num_filters=3, act='sigmoid', output_size=[image_size, image_size])
29    return y

判别器的作用是训练真实的数据集，然后使用训练真实数据集模型去判别生成器生成的假图片。这一过程可以理解判别器为一个二分类问题，判别器在训练真实数据集时，尽量让其输出概率为1，而训练生成器生成的假图片输出概率为0。这样不断给生成器压力，让其生成的图片尽量逼近真实图片，以至于真实到连判别器也无法判断这是真实图像还是假图片。以下判别器由三组卷积池化层和一个最后全连接层组成，全连接层的大小为1，输入一个二分类的结果。

 1# 判别器 Discriminator
 2def Discriminator(images, name="D"):
 3    # 定义一个卷积池化组
 4    def conv_pool(input, num_filters, act=None):
 5        return fluid.nets.simple_img_conv_pool(input=input,
 6                                               filter_size=3,
 7                                               num_filters=num_filters,
 8                                               pool_size=2,
 9                                               pool_stride=2,
10                                               act=act)
11
12    with fluid.unique_name.guard(name + "/"):
13        y = fluid.layers.reshape(x=images, shape=[-1, 3, image_size, image_size])
14        # 第一个卷积池化组
15        y = conv_pool(input=y, num_filters=64, act='leaky_relu')
16        # 第一个卷积池化加回归层
17        y = conv_pool(input=y, num_filters=128)
18        y = fluid.layers.batch_norm(input=y, act='leaky_relu')
19        # 第二个卷积池化加回归层
20        y = fluid.layers.fc(input=y, size=1024)
21        y = fluid.layers.batch_norm(input=y, act='leaky_relu')
22        # 最后一个分类器输出
23        y = fluid.layers.fc(input=y, size=1, act='sigmoid')
24    return y

然后在这里获取所需的程序，如判别器D识别生成器G生成的假图片程序，判别器D识别真实图片程序，生成器G生成符合判别器D的程序和初始化的程序。最后定义一个get_params()函数用于获取参数名称。

 1# 创建判别器D识别生成器G生成的假图片程序
 2train_d_fake = fluid.Program()
 3# 创建判别器D识别真实图片程序
 4train_d_real = fluid.Program()
 5# 创建生成器G生成符合判别器D的程序
 6train_g = fluid.Program()
 7
 8# 创建共同的一个初始化的程序
 9startup = fluid.Program()
10
11# 噪声维度
12z_dim = 100
13
14# 从Program获取prefix开头的参数名字
15def get_params(program, prefix):
16    all_params = program.global_block().all_parameters()
17    return [t.name for t in all_params if t.name.startswith(prefix)]

定义一个判别器识别真实图片的程序，这里判别器传入的数据是真实的图片数据，这里的输出图片是3通道的。这里使用的损失函数是fluid.layers.sigmoid_cross_entropy_with_logits()，这个损失函数是求它们在任务上的错误率，他们的类别是互不排斥的。所以无论真实图片的标签是什么，都不会影响模型识别为真实图片。这里更新的也只有判别器模型的参数，使用的优化方法是Adam。

 1# 训练判别器D识别真实图片
 2with fluid.program_guard(train_d_real, startup):
 3    # 创建读取真实数据集图片的data，并且label为1
 4    real_image = fluid.layers.data('image', shape=[3, image_size, image_size])
 5    ones = fluid.layers.fill_constant_batch_size_like(real_image, shape=[-1, 1], dtype='float32', value=1)
 6
 7    # 判别器D判断真实图片的概率
 8    p_real = Discriminator(real_image)
 9    # 获取损失函数
10    real_cost = fluid.layers.sigmoid_cross_entropy_with_logits(p_real, ones)
11    real_avg_cost = fluid.layers.mean(real_cost)
12
13    # 获取判别器D的参数
14    d_params = get_params(train_d_real, "D")
15
16    # 创建优化方法
17    optimizer = fluid.optimizer.Adam(learning_rate=2e-4)
18    optimizer.minimize(real_avg_cost, parameter_list=d_params)

这里定义一个判别器识别生成器生成的图片的程序，这里是使用噪声的维度进行输入。这里判别器识别的是生成器生成的图片，这里使用的损失函数同样是fluid.layers.sigmoid_cross_entropy_with_logits()。这里更新的参数还是判别器模型的参数，也是使用Adam优化方法。

 1# 训练判别器D识别生成器G生成的图片为假图片
 2with fluid.program_guard(train_d_fake, startup):
 3    # 利用创建假的图片data，并且label为0
 4    z = fluid.layers.data(name='z', shape=[z_dim])
 5    zeros = fluid.layers.fill_constant_batch_size_like(z, shape=[-1, 1], dtype='float32', value=0)
 6
 7    # 判别器D判断假图片的概率
 8    p_fake = Discriminator(Generator(z))
 9
10    # 获取损失函数
11    fake_cost = fluid.layers.sigmoid_cross_entropy_with_logits(p_fake, zeros)
12    fake_avg_cost = fluid.layers.mean(fake_cost)
13
14    # 获取判别器D的参数
15    d_params = get_params(train_d_fake, "D")
16
17    # 创建优化方法
18    optimizer = fluid.optimizer.Adam(learning_rate=2e-4)
19    optimizer.minimize(fake_avg_cost, parameter_list=d_params)

最后定义一个训练生成器生成图片的模型，这里也克隆一个预测程序，用于之后在训练的时候输出预测的图片。损失函数和优化方法都一样，但是要更新的参数是生成器的模型参。

 1# 训练生成器G生成符合判别器D标准的假图片
 2fake = None
 3with fluid.program_guard(train_g, startup):
 4    # 噪声生成图片为真实图片的概率，Label为1
 5    z = fluid.layers.data(name='z', shape=[z_dim])
 6    ones = fluid.layers.fill_constant_batch_size_like(z, shape=[-1, 1], dtype='float32', value=1)
 7
 8    # 生成图片
 9    fake = Generator(z)
10    # 克隆预测程序
11    infer_program = train_g.clone(for_test=True)
12
13    # 生成符合判别器的假图片
14    p = Discriminator(fake)
15
16    # 获取损失函数
17    g_cost = fluid.layers.sigmoid_cross_entropy_with_logits(p, ones)
18    g_avg_cost = fluid.layers.mean(g_cost)
19
20    # 获取G的参数
21    g_params = get_params(train_g, "G")
22
23    # 只训练G
24    optimizer = fluid.optimizer.Adam(learning_rate=2e-4)
25    optimizer.minimize(g_avg_cost, parameter_list=g_params)

这里创建一个可以生成训练噪声数据的reader函数。

1# 噪声生成
2def z_reader():
3    while True:
4        yield np.random.uniform(-1.0, 1.0, (z_dim)).astype('float32')

这里定义一个保存在训练过程生成的图片，通过观察生成图片的情况，可以了解到训练的效果。

1# 保存图片
2def show_image_grid(images):
3    for i, image in enumerate(images):
4        image = image.transpose((2, 1, 0))
5        save_image_path = 'train_image'
6        if not os.path.exists(save_image_path):
7            os.makedirs(save_image_path)
8        plt.imsave(os.path.join(save_image_path, "test_%d.png" % i), image)

这里就开始获取自定义的图片数据集，这里只需要把存放图片数据集的文件夹传进去就可以了。

1# 生成真实图片reader
2mydata_generator = paddle.batch(reader=image_reader.train_reader('datasets', image_size), batch_size=32)
3# 生成假图片的reader
4z_generator = paddle.batch(z_reader, batch_size=32)()
5test_z = np.array(next(z_generator))

接着获取执行器，准备进行训练，这里笔者建议最好使用GPU，因为CPU贼慢。

1# 创建执行器，最好使用GPU，CPU速度太慢了
2# place = fluid.CPUPlace()
3place = fluid.CUDAPlace(0)
4exe = fluid.Executor(place)
5# 初始化参数
6exe.run(startup)

最好就可以开始训练啦，我们可以在训练的时候输出训练的损失值。在训练每一个Pass之后又可以使用预测程序生成图片并进行保存到本地。

 1# 开始训练
 2for pass_id in range(100):
 3    for i, real_image in enumerate(mydata_generator()):
 4        # 训练判别器D识别真实图片
 5        r_fake = exe.run(program=train_d_fake,
 6                         fetch_list=[fake_avg_cost],
 7                         feed={'z': test_z})
 8
 9        # 训练判别器D识别生成器G生成的假图片
10        r_real = exe.run(program=train_d_real,
11                         fetch_list=[real_avg_cost],
12                         feed={'image': np.array(real_image)})
13
14        # 训练生成器G生成符合判别器D标准的假图片
15        r_g = exe.run(program=train_g,
16                      fetch_list=[g_avg_cost],
17                      feed={'z': test_z})
18
19        if i % 100 == 0:
20            print("Pass：%d, Batch：%d, 训练判别器D识别真实图片Cost：%0.5f, "
21                  "训练判别器D识别生成器G生成的假图片Cost：%0.5f, "
22                  "训练生成器G生成符合判别器D标准的假图片Cost：%0.5f" % (pass_id, i, r_fake[0], r_real[0], r_g[0]))
23
24    # 测试生成的图片
25    r_i = exe.run(program=infer_program,
26                  fetch_list=[fake],
27                  feed={'z': test_z})
28
29    r_i = np.array(r_i).astype(np.float32)
30    # 显示生成的图片

同时在每个Pass之后又可以保存预测函数，用于之后预测生成图片使用。

使用模型生成图片

在上一个文件中，我们已经训练得到一个预测模型，下面我们将使用这个预测模型直接生成图片。创建infer.py文件用于预测生成图片。首先导入相应的依赖包。

1import os
2import paddle
3import matplotlib.pyplot as plt
4import numpy as np
5import paddle.fluid as fluid

然后创建执行器，这里可以使用CPU进行预测可以，因为预测并不需要太大的计算。然后加载上一步训练保存的预测模型，获取预测程序，输入层的名称，和生成器。

1# 创建执行器
2place = fluid.CPUPlace()
3exe = fluid.Executor(place)
4exe.run(fluid.default_startup_program())
5
6# 保存预测模型路径
7save_path = 'infer_model/'
8# 从模型中获取预测程序、输入数据名称列表、分类器
9[infer_program, feeded_var_names, target_var] = fluid.io.load_inference_model(dirname=save_path, executor=exe)

跟训练的时候一样，需要生成噪声数据作为输入数据。这里说明一下，输入数据z_generator 的batch大小就是生成图片的数量。

 1# 噪声维度
 2z_dim = 100
 3
 4# 噪声生成
 5def z_reader():
 6    while True:
 7        yield np.random.uniform(-1.0, 1.0, (z_dim)).astype('float32')
 8
 9z_generator = paddle.batch(z_reader, batch_size=32)()
10test_z = np.array(next(z_generator))

这里创建一个保存生成图片的函数，用于保存预测生成的图片。

1# 保存图片
2def save_image(images):
3    for i, image in enumerate(images):
4        image = image.transpose((2, 1, 0))
5        save_image_path = 'infer_image'

最后执行预测程序，开始生成图片。预测输出的结果就是图片的数据，通过保存这些数据就是保存图片了。

 1# 测试生成的图片
 2r_i = exe.run(program=infer_program,
 3              feed={feeded_var_names[0]: test_z},

目前这个网络在训练比较复杂的图片时，模型的拟合效果并不太好，也就是说生成的图片没有我们想象那么好。所以这个网络还需要不断调整，如果读者有更好的建议，欢迎交流一下。
