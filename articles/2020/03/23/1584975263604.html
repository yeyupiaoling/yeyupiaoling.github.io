前言

现在越来越多的手机要使用到深度学习了，比如一些图像分类，目标检测，风格迁移等等，之前都是把数据提交给服务器完成的。但是提交给服务器有几点不好，首先是速度问题，图片上传到服务器需要时间，客户端接收结果也需要时间，这一来回就占用了一大半的时间，会使得整体的预测速度都变慢了，再且现在手机的性能不断提高，足以做深度学习的预测。其二是隐私问题，如果只是在本地预测，那么用户根本就不用上传图片，安全性也大大提高了。所以本章我们就来学如何包我们训练的PaddlePaddle预测模型部署到Android手机上。

编译paddle-mobile库

想要把PaddlePaddle训练好的预测库部署到Android手机上，还需要借助paddle-mobile框架。paddle-mobile框架主要是为了方便PaddlePaddle训练好的模型部署到移动设备上，比如Android手机，苹果手机，树莓派等等这些移动设备，有了paddle-mobile框架大大方便了把PaddlePaddle的预测库部署到移动设备上，而且paddle-mobile框架针对移动设备做了大量的优化，使用这些预测库在移动设备上有了更好的预测性能。

想要在Android手机上使用paddle-mobile，就要编译Android能够使用的CPP库，在这一部分中，我们介绍两种编译Android的paddle-mobile库，分别是使用Docker编译paddle-mobile库、使用Ubuntu交叉编译paddle-mobile库。

使用Docker编译

为了方便操作，以下的操作都是在root用户的执行的：

  1. 安装Docker，以下是在Ubuntu下安装的的方式，只要一条命令就可以了：
1apt-get install docker.io
  1. 克隆paddle-mobile源码：
1git clone https://github.com/PaddlePaddle/paddle-mobile.git
  1. 进入到paddle-mobile根目录下编译docker镜像：
1cd paddle-mobile
2# 编译生成进行，编译时间可能要很长
3docker build -t paddle-mobile:dev - < Dockerfile

编译完成可以使用 docker images命令查看是否已经生成进行：

1docker run -it -v $PWD:/paddle-mobile paddle-mobile:dev
  1. 在容器里面执行以下两条命令：

  1. 然后在项目的主要包下创建一个 Utils.java的工具类。这个工具类主要编写一些图像的处理方法，和一些模型复制方法等，我们下面将一一介绍这些方法。

该方法是用于获取预测结果中概率最大的标签，参数是执行预测的结果，这个结果是对应没有类别的概率，这个方法就判断哪个类别的概率最大，然后就返回概率最大的标签。

 1// 获取预测值中最大概率的标签
 2public static int getMaxResult(float[] result) {
 3    float probability = result[0];
 4    int r = 0;
 5    for (int i = 0; i < result.length; i++) {
 6        if (probability < result[i]) {
 7            probability = result[i];
 8            r = i;
 9        }
10    }
11    return r;
12}

该方法是把图片转换成预测需要用的数据格式浮点数组。在转换的过程中也对图像做了预处理，这个预处理需要跟训练的预处理的方式一样，否则无法正确预测。还有指定了处理后图片的大小，根据参数输入的宽度和高度，把图片压缩到这些自定的大小。还有把图片的通道顺序改为RGB，同时每个像素除以255，这个操作跟训练的时候一样。

 1// 对将要预测的图片进行预处理
 2public static float[] getScaledMatrix(Bitmap bitmap, int desWidth, int desHeight) {
 3    float[] dataBuf = new float[3 * desWidth * desHeight];
 4    int rIndex;
 5    int gIndex;
 6    int bIndex;
 7    int[] pixels = new int[desWidth * desHeight];
 8    Bitmap bm = Bitmap.createScaledBitmap(bitmap, desWidth, desHeight, false);
 9    bm.getPixels(pixels, 0, desWidth, 0, 0, desWidth, desHeight);
10    int j = 0;
11    int k = 0;
12    for (int i = 0; i < pixels.length; i++) {
13        int clr = pixels[i];
14        j = i / desHeight;
15        k = i % desWidth;
16        rIndex = j * desWidth + k;
17        gIndex = rIndex + desHeight * desWidth;
18        bIndex = gIndex + desHeight * desWidth;
19        // 转成RGB通道顺序，并除以255，跟训练的预处理一样
20        dataBuf[rIndex] = (float) (((clr & 0x00ff0000) >> 16) / 255.0);
21        dataBuf[gIndex] = (float) (((clr & 0x0000ff00) >> 8) / 255.0);
22        dataBuf[bIndex] = (float) (((clr & 0x000000ff)) / 255.0);
23
24    }
25    if (bm.isRecycled()) {
26        bm.recycle();
27    }
28    return dataBuf;
29}

该方法是对图片进行压缩，避免图片过大，超过内存支出。把图片的最大长度压缩到500以内。

 1// 压缩图片，避免图片过大
 2public static Bitmap getScaleBitmap(String filePath) {
 3    BitmapFactory.Options opt = new BitmapFactory.Options();
 4    opt.inJustDecodeBounds = true;
 5    BitmapFactory.decodeFile(filePath, opt);
 6
 7    int bmpWidth = opt.outWidth;
 8    int bmpHeight = opt.outHeight;
 9
10    int maxSize = 500;
11
12    // compress picture with inSampleSize
13    opt.inSampleSize = 1;
14    while (true) {
15        if (bmpWidth / opt.inSampleSize < maxSize || bmpHeight / opt.inSampleSize < maxSize) {
16            break;
17        }
18        opt.inSampleSize *= 2;
19    }
20    opt.inJustDecodeBounds = false;
21    return BitmapFactory.decodeFile(filePath, opt);
22}

该方法是根据相册返回的URI转换为图片的绝对路径，用于之后使用这个路径获取图片内容。

 1// 根据相册返回的URI返回图片的绝对路径
 2public static String getPathFromURI(Context context, Uri uri) {
 3    String result;
 4    Cursor cursor = context.getContentResolver().query(uri, null, null, null, null);
 5    if (cursor == null) {
 6        result = uri.getPath();
 7    } else {
 8        cursor.moveToFirst();
 9        int idx = cursor.getColumnIndex(MediaStore.Images.ImageColumns.DATA);
10        result = cursor.getString(idx);
11        cursor.close();
12    }
13    return result;
14}

该方法是把 assets资源文件下的预测文件复制到缓存目录，用于之后加载模型文件。

 1// 复制莫模型文件到缓存目录
 2public static void copyFileFromAsset(Context context, String oldPath, String newPath) {
 3    try {
 4        // 预测模型文件在assets中的位置
 5        String[] fileNames = context.getAssets().list(oldPath);
 6        if (fileNames.length > 0) {
 7            // directory
 8            File file = new File(newPath);
 9            if (!file.exists()) {
10                file.mkdirs();
11            }
12            // copy recursivelyC
13            for (String fileName : fileNames) {
14                copyFileFromAsset(context, oldPath + "/" + fileName, newPath + "/" + fileName);
15            }
16        } else {
17            // file
18            File file = new File(newPath);
19            // if file exists will never copy
20            if (file.exists()) {
21                return;
22            }
23
24            // copy file to new path
25            InputStream is = context.getAssets().open(oldPath);
26            FileOutputStream fos = new FileOutputStream(file);
27            byte[] buffer = new byte[1024];
28            int byteCount;
29            while ((byteCount = is.read(buffer)) != -1) {
30                fos.write(buffer, 0, byteCount);
31            }
32            fos.flush();
33            is.close();
34            fos.close();
35        }
36    } catch (Exception e) {
37        e.printStackTrace();
38    }
39}

该方法是一个回调方法，主要是打开相册后的回调预测操作。使用返回的URI转换为绝对路径，然后使用这个图片路径转换成Bitmap用于显示，同时也使用这个路径执行预测操作。

该方法是预测操作的方法，参数是图片的绝对路径，首先根据图片获取已经压缩过的Bitmap，然后使用这个Bitmap转换成预处理后的浮点数组，最后执行预测操作。再根据预测结果提取最大概率的标签，并获取该标签的类别名称。

 1    // 根据图片的路径预测图片
 2    private void predictImage(String image_path) {
 3        // 把图片进行压缩
 4        Bitmap bmp = Utils.getScaleBitmap(image_path);
 5        // 把图片转换成浮点数组，用于预测
 6        float[] inputData = Utils.getScaledMatrix(bmp, ddims[2], ddims[3]);
 7        try {
 8            long start = System.currentTimeMillis();
 9            // 执行预测，获取预测结果
10            float[] result = PML.predictImage(inputData, ddims);
11            long end = System.currentTimeMillis();
12            // 获取概率最大的标签
13            int r = Utils.getMaxResult(result);
14            // 获取标签对应的类别名称
15            String[] names = {"苹果", "哈密瓜", "胡萝卜", "樱桃", "黄瓜", "西瓜"};
16            String show_text = "标签：" + r + "\n名称：" + names[r] + "\n概率：" + result[r] + "\n时间：" + (end - start) + "ms";
17            // 显示预测结果
18            showTv.setText(show_text);
19        } catch (Exception e) {
20            e.printStackTrace();
21        }
22    }
