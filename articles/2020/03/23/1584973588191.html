在Android手机上使用腾讯的ncnn实现图像分类

2020-03-23
   
Android 人工智能 夜雨飘零 深度学习
    0 0

前言

在之前笔者有介绍过《在Android设备上使用PaddleMobile实现图像分类》，使用的框架是百度开源的PaddleMobile。在本章中，笔者将会介绍使用腾讯的开源手机深度学习框架ncnn来实现在Android手机实现图像分类，这个框架开源时间比较长，相对稳定很多。

ncnn的GitHub地址：https://github.com/Tencent/ncnn

使用Ubuntu编译ncnn库

1、首先要下载和解压NDK。

1wget https://dl.google.com/android/repository/android-ndk-r17b-linux-x86_64.zip
2unzip android-ndk-r17b-linux-x86_64.zip

2、设置NDK环境变量，目录是NDK的解压目录。

1export NDK_ROOT="/home/test/paddlepaddle/android-ndk-r17b"

设置好之后，可以使用以下的命令查看配置情况。

1root@test:/home/test/paddlepaddle# echo $NDK_ROOT
2/home/test/paddlepaddle/android-ndk-r17b

3、安装cmake，需要安装较高版本的，笔者的cmake版本是3.11.2。

下载cmake源码

1wget https://cmake.org/files/v3.11/cmake-3.11.2.tar.gz

解压cmake源码

1tar -zxvf cmake-3.11.2.tar.gz

进入到cmake源码根目录，并执行bootstrap。

1cd cmake-3.11.2
2./bootstrap

最后执行以下两条命令开始安装cmake。

1make
2make install

安装完成之后，可以使用cmake --version是否安装成功。

1root@test:/home/test/paddlepaddle# cmake --version
2cmake version 3.11.2
3
4CMake suite maintained and supported by Kitware (kitware.com/cmake).

4、克隆ncnn源码。

1git clone https://github.com/Tencent/ncnn.git

5、编译源码。

 1# 进入到ncnn源码根目录下
 2cd ncnn
 3# 创建一个新的文件夹
 4mkdir -p build-android-armv7
 5# 进入到该文件夹中
 6cd build-android-armv7
 7# 执行编译命令
 8cmake -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK/build/cmake/android.toolchain.cmake \
 9    -DANDROID_ABI="armeabi-v7a" -DANDROID_ARM_NEON=ON \
10    -DANDROID_PLATFORM=android-14 ..
11# 这里笔者使用4个行程并行编译
12make -j4
13make install

6、编译完成，会在build-android-armv7目录下生成一个install目录，我们编译得到的文件都在该文件夹下：

  • include 调用ncnn所需的头文件，该文件夹会存放在Android项目的src/main/cpp目录下；
  • lib 编译得到的ncnn库libncnn.a，之后会存放在Android项目的src/main/jniLibs/armeabi-v7a/libncnn.a

转换预测模型

1、克隆Caffe源码。

1git clone https://github.com/BVLC/caffe.git

2、编译Caffe源码。

1# 切换到Caffe目录
2cd caffe
3# 在当前目录执行cmake
4cmake .
5# 使用4个线程编译
6make -j4
7make install

3、升级Caffe模型。

1# 把需要转换的模型复制到caffe/tools，并切入到该目录
2cd tools
3# 升级Caffe模型
4./upgrade_net_proto_text mobilenet_v2_deploy.prototxt mobilenet_v2_deploy_new.prototxt
5./upgrade_net_proto_binary mobilenet_v2.caffemodel mobilenet_v2_new.caffemodel

4、检查模型配置文件，因为只能一张一张图片预测，所以输入要设置为dim: 1。

 1name: "MOBILENET_V2"
 2layer {
 3  name: "input"
 4  type: "Input"
 5  top: "data"
 6  input_param {
 7    shape {
 8      dim: 1
 9      dim: 3
10      dim: 224
11      dim: 224
12    }
13  }
14}

5、切换到ncnn的根目录，就是我们上一部分克隆的ncnn源码。

1cd ncnn/

6、在根目录下编译ncnn源码。

1mkdir -p build
2cd build
3cmake ..
4make -j4
5make install

7、把新的Caffe模型转换成NCNN模型。

1# 经过上一步，会生产一个tools，我们进入到以下目录
2cd tools/caffe/
3# 把已经升级的网络定义文件和权重文件复制到当目录，并执行以下命令
4./caffe2ncnn mobilenet_v2_deploy_new.prototxt mobilenet_v2_new.caffemodel mobilenet_v2.param mobilenet_v2.bin

8、对象模型参数进行加密，这样就算别人反编译我们的apk也用不了我们的模型文件。把上一步获得的mobilenet_v2.param、mobilenet_v2.bin复制到该目录的上一个目录，也就是tools目录。

1# 切换到上一个目录
2cd ../
3# 执行命令之后会生成mobilenet_v2.param、mobilenet_v2.id.h、mobilenet_v2.mem.h
4./ncnn2mem mobilenet_v2.param mobilenet_v2.bin mobilenet_v2.id.h mobilenet_v2.mem.h

经过上面的步骤，得到的文件中，以下文件时需要的：

  • mobilenet_v2.param.bin 网络的模型参数；
  • mobilenet_v2.bin 网络的权重；
  • mobilenet_v2.id.h 在预测图片的时候使用到。

 54// public native String Detect(Bitmap bitmap);
 55JNIEXPORT jfloatArray JNICALL Java_com_example_ncnn1_NcnnJni_Detect(JNIEnv* env, jobject thiz, jobject bitmap)
 56{
 57    // ncnn from bitmap
 58    ncnn::Mat in;
 59    {
 60        AndroidBitmapInfo info;
 61        AndroidBitmap_getInfo(env, bitmap, &info);
 62        int width = info.width;
 63        int height = info.height;
 64        if (info.format != ANDROID_BITMAP_FORMAT_RGBA_8888)
 65            return NULL;
 66
 67        void* indata;
 68        AndroidBitmap_lockPixels(env, bitmap, &indata);
 69        // 把像素转换成data，并指定通道顺序
 70        in = ncnn::Mat::from_pixels((const unsigned char*)indata, ncnn::Mat::PIXEL_RGBA2BGR, width, height);
 71
 72        AndroidBitmap_unlockPixels(env, bitmap);
 73    }
 74
 75    // ncnn_net
 76    std::vector<float> cls_scores;
 77    {
 78        // 减去均值和乘上比例
 79        const float mean_vals[3] = {103.94f, 116.78f, 123.68f};
 80        const float scale[3] = {0.017f, 0.017f, 0.017f};
 81
 82        in.substract_mean_normalize(mean_vals, scale);
 83
 84        ncnn::Extractor ex = ncnn_net.create_extractor();
 85        // 如果时不加密是使用ex.input("data", in);
 86        ex.input(mobilenet_v2_param_id::BLOB_data, in);
 87
 88        ncnn::Mat out;
 89        // 如果时不加密是使用ex.extract("prob", out);
 90        ex.extract(mobilenet_v2_param_id::BLOB_prob, out);
 91
 92        int output_size = out.w;
 93        jfloat *output[output_size];
 94        for (int j = 0; j < out.w; j++) {
 95            output[j] = &out[j];
 96        }
 97
 98        jfloatArray jOutputData = env->NewFloatArray(output_size);
 99        if (jOutputData == nullptr) return nullptr;
100        env->SetFloatArrayRegion(jOutputData, 0, output_size,
101                                 reinterpret_cast<const jfloat *>(*output));  // copy
102
103        return jOutputData;
104    }
105}
