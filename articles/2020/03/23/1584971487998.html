《我的PaddlePaddle学习之路》笔记十四——把PaddlePaddle迁移到Android设备上

2020-03-23
   
PaddlePaddle 人工智能 夜雨飘零 深度学习
    0 0

*本篇文章基于 PaddlePaddle 0.11.0、Python 2.7

前言

PaddlePaddle还可以迁移到Android或者Linux设备上，在这些部署了PaddlePaddle的设备同样可以做深度学习的预测。在这篇文章中我们就介绍如何把PaddlePaddle迁移到Android手机上，并在Android的APP中使用PaddlePaddle。

编译PaddlePaddle库

使用Docker编译PaddlePaddle库

使用Docker编译PaddlePaddle真的会很方便，如果你对比了下面部分的使用Linux编译PaddlePaddle库，你就会发现使用Docker会少很多麻烦，比如安装一些依赖库等等。而且Docker是跨平台的，不管读者使用的是Windows，Linux，还是Mac，都可以使用Docker。以下操作方法都是在64位的Ubuntu 16.04上实现的。
首先安装Docker，在Ubuntu上安装很简单，只要一条命令就可以了

1sudo apt install docker.io

安装完成之后，可以使用docker --version命令查看是否安装成功，如果安装成功会输出Docker的版本信息。

然后是在GitHub上克隆PaddlePaddle源码，命令如下：

1git clone https://github.com/PaddlePaddle/Paddle.git

克隆完成PaddlePaddle源码之后，就可以使用PaddlePaddle源码创建可以编译给Android使用的PaddlePaddle库的Docker容器了：

1# 切入到源码目录
2cd Paddle
3# 创建Docker容器
4docker build -t mypaddle/paddle-android:dev . -f Dockerfile.android

可能会出现的问题

值得注意的是如果读者的电脑不能科学上网的，会在下载https://storage.googleapis.com/golang/go1.8.1.linux-amd64.tar.gz的时候报错，可以修改其下载路径。使用的文件是在Paddle目录下的Dockerfile.android，所以要在这个文件中修改，具体在25行，将其修改成https://dl.google.com/go/go1.8.1.linux-amd64.tar.gz，如果可以科学上网，就不必理会。
如果再不行就干脆去掉GO语言依赖，因为编译Android的PaddlePaddle库根本就不用GO语言依赖库，具体操作如下：
修改Paddle/CMakeLists.txt下的22行

1 project(paddle CXX C Go)

去掉Go依赖，修改成如下

1project(paddle CXX C)

删除Paddle/Dockerfile.android的Go语言配置

1# Install Go and glide
2RUN wget -qO- go.tgz https://storage.googleapis.com/golang/go1.8.1.linux-amd64.tar.gz | \
3    tar -xz -C /usr/local && \
4    mkdir /root/gopath && \
5    mkdir /root/gopath/bin && \
6    mkdir /root/gopath/src
7ENV GOROOT=/usr/local/go GOPATH=/root/gopath
8# should not be in the same line with GOROOT definition, otherwise docker build could not find GOROOT.
9ENV PATH=${PATH}:${GOROOT}/bin:${GOPATH}/bin

使用官方的Docker容器

如果读者不想使用源码创建Docker容器，PaddlePaddle官方也提供了创建好的Docker容器，读者可以直接拉到本地就可以使用了，命令如下：

1docker pull paddlepaddle/paddle:latest-dev-android

以上是国外的镜像，如果pull的速度慢，可以使用国内的镜像

1docker pull docker.paddlepaddlehub.com/paddle:latest-dev-android

开始编译PaddlePaddle库

编译armeabi-v7a，Android API 21的PaddlePaddle库，命令如下，创建PaddlePaddle的配置可以使用e命令设置。在命令的最后可以看到使用的容器是我们自己创建的Docker容器mypaddle/paddle-android:dev，如果换成官方提供的，把Docker名称修改成paddlepaddle/paddle:latest-dev-android即可。

1docker run -it --rm -v $PWD:/paddle -e "ANDROID_ABI=armeabi-v7a" -e "ANDROID_API=21" mypaddle/paddle-android:dev

当编译完成之后，在$PWD/install_android目录下创建以下三个目录，$PWD表示当前目录，笔者当前目录为/home/work/android/docker/。这些文件就是我们之后在Android的APP上会使用的的文件：

  • include是C-API的头文件
  • lib是Android ABI的PaddlePaddle库
  • third_party是所依赖的所有第三方库

上面的是编译armeabi-v7a，Android API 21的PaddlePaddle库，如果读者想编译arm64-v8a，Android API 21的PaddlePaddle库，只要修改命令参数就可以了，具体命令如下：

1docker run -it --rm -v $PWD:/paddle -e "ANDROID_ABI=arm64-v8a" -e "ANDROID_API=21" mypaddle/paddle-android:dev

如果读者不习惯与使用Docker，或者想进一步了解编译PaddlePaddle库的流程，想使用Linux编译PaddlePaddle库，这也是没有问题的，只是步骤比较复杂一些。

训练模型

我们要使用PaddlePadad预先训练我们的神经网络模型才能进行下一步操作。我们这次使用的是mobilenet神经网络，这个网络更它的名字一样，是为了移植到移动设备上的一个神经网络，虽然我们第三章的CIFAR彩色图像识别使用的是VGG神经模型，但是使用的流程基本上是一样的。

定义神经网络

创建一个mobilenet.py的Python文件，来定义我的mobilenet神经网络模型。mobilenet是Google针对手机等嵌入式设备提出的一种轻量级的深层神经网络，它的核心思想就是卷积核的巧妙分解，可以有效减少网络参数，从而达到减小训练时网络的模型。因为太大的模型参数是不利于移植到移动设备上的，比如我们使用的VGG在训练CIFAR10的时候，模型会有58M那么大，这样的模型如下移植到Android应用上，那会大大增加apk的大小，这样是不利于应用的推广的。

  1# edit-mode: -*- python -*-
  2import paddle.v2 as paddle

  5def conv_bn_layer(input,
  6                  filter_size,
  7                  num_filters,
  8                  stride,
  9                  padding,
 10                  channels=None,
 11                  num_groups=1,
 12                  active_type=paddle.activation.Relu(),
 13                  layer_type=None):
 14    """
 15    A wrapper for conv layer with batch normalization layers.
 16    Note:
 17    conv layer has no activation.
 18    """
 19    tmp = paddle.layer.img_conv(
 20        input=input,
 21        filter_size=filter_size,
 22        num_channels=channels,
 23        num_filters=num_filters,
 24        stride=stride,
 25        padding=padding,
 26        groups=num_groups,
 27        act=paddle.activation.Linear(),
 28        bias_attr=False,
 29        layer_type=layer_type)
 30    return paddle.layer.batch_norm(input=tmp, act=active_type)
 31
 32
 33def depthwise_separable(input, num_filters1, num_filters2, num_groups, stride,
 34                        scale):
 35    """
 36    """
 37    tmp = conv_bn_layer(
 38        input=input,
 39        filter_size=3,
 40        num_filters=int(num_filters1 * scale),
 41        stride=stride,
 42        padding=1,
 43        num_groups=int(num_groups * scale),
 44        layer_type='exconv')
 45
 46    tmp = conv_bn_layer(
 47        input=tmp,
 48        filter_size=1,
 49        num_filters=int(num_filters2 * scale),
 50        stride=1,
 51        padding=0)
 52    return tmp
 53

 55def mobile_net(img_size, class_num, scale=1.0):
 56
 57    img = paddle.layer.data(
 58        name="image", type=paddle.data_type.dense_vector(img_size))
 59
 60    # conv1: 112x112
 61    tmp = conv_bn_layer(
 62        img,
 63        filter_size=3,
 64        channels=3,
 65        num_filters=int(32 * scale),
 66        stride=2,
 67        padding=1)
 68
 69    # 56x56
 70    tmp = depthwise_separable(
 71        tmp,
 72        num_filters1=32,
 73        num_filters2=64,
 74        num_groups=32,
 75        stride=1,
 76        scale=scale)
 77    tmp = depthwise_separable(
 78        tmp,
 79        num_filters1=64,
 80        num_filters2=128,
 81        num_groups=64,
 82        stride=2,
 83        scale=scale)
 84    # 28x28
 85    tmp = depthwise_separable(
 86        tmp,
 87        num_filters1=128,
 88        num_filters2=128,
 89        num_groups=128,
 90        stride=1,
 91        scale=scale)
 92    tmp = depthwise_separable(
 93        tmp,
 94        num_filters1=128,
 95        num_filters2=256,
 96        num_groups=128,
 97        stride=2,
 98        scale=scale)
 99    # 14x14
100    tmp = depthwise_separable(
101        tmp,
102        num_filters1=256,
103        num_filters2=256,
104        num_groups=256,
105        stride=1,
106        scale=scale)
107    tmp = depthwise_separable(
108        tmp,
109        num_filters1=256,
110        num_filters2=512,
111        num_groups=256,
112        stride=2,
113        scale=scale)
114    # 14x14
115    for i in range(5):
116        tmp = depthwise_separable(
117            tmp,
118            num_filters1=512,
119            num_filters2=512,
120            num_groups=512,
121            stride=1,
122            scale=scale)
123    # 7x7
124    tmp = depthwise_separable(
125        tmp,
126        num_filters1=512,
127        num_filters2=1024,
128        num_groups=512,
129        stride=2,
130        scale=scale)
131    tmp = depthwise_separable(
132        tmp,
133        num_filters1=1024,
134        num_filters2=1024,
135        num_groups=1024,
136        stride=1,
137        scale=scale)
138
139    tmp = paddle.layer.img_pool(
140        input=tmp, pool_size=7, stride=1, pool_type=paddle.pooling.Avg())
141    out = paddle.layer.fc(
142        input=tmp, size=class_num, act=paddle.activation.Softmax())
143
144    return out

获取训练参数

然后是编写获取训练参数的代码，这个提供了两个获取参数的方法，一个是从损失函数中创建一个训练参数，另一个是使用之前训练好的训练参数：

 1def get_parameters(self, parameters_path=None, cost=None):
 2    if not parameters_path:
 3        # 使用cost创建parameters
 4        if not cost:
 5            print "请输入cost参数"
 6        else:
 7            # 根据损失函数创建参数
 8            parameters = paddle.parameters.create(cost)
 9            return parameters
10    else:
11        # 使用之前训练好的参数
12        try:
13            # 使用训练好的参数
14            with gzip.open(parameters_path, 'r') as f:
15                parameters = paddle.parameters.Parameters.from_tar(f)
16            return parameters
17        except Exception as e:
18            raise NameError("你的参数文件错误,具体问题是:%s" % e)

获取训练器

通过损失函数、训练参数、优化方法可以创建一个训练器。

  • cost，损失函数，通过神经网络的分类器和分类的标签可以获取损失函数。
  • parameters，训练参数，这个在上已经讲过了，这里就不重复了。
  • optimizer，优化方法，这个优化方法是设置学习率和加正则的。
 1def get_trainer(self):
 2    # 数据大小
 3    datadim = 3 * 32 * 32
 4
 5    # 获得图片对于的信息标签
 6    lbl = paddle.layer.data(name="label",
 7                            type=paddle.data_type.integer_value(10))
 8
 9    # 获取全连接层,也就是分类器
10    out = mobile_net(datadim, 10, 1.0)
11
12    # 获得损失函数
13    cost = paddle.layer.classification_cost(input=out, label=lbl)
14
15    # 使用之前保存好的参数文件获得参数
16    # parameters = self.get_parameters(parameters_path="../model/mobile_net.tar.gz")
17    # 使用损失函数生成参数
18    parameters = self.get_parameters(cost=cost)
19
20    '''
21    定义优化方法
22    learning_rate 迭代的速度
23    momentum 跟前面动量优化的比例
24    regularzation 正则化,防止过拟合
25    '''
26    momentum_optimizer = paddle.optimizer.Momentum(
27        momentum=0.9,
28        regularization=paddle.optimizer.L2Regularization(rate=0.0002 * 128),
29        learning_rate=0.1 / 128.0,
30        learning_rate_decay_a=0.1,
31        learning_rate_decay_b=50000 * 100,
32        learning_rate_schedule="discexp")
33
34    '''
35    创建训练器
36    cost 分类器
37    parameters 训练参数,可以通过创建,也可以使用之前训练好的参数
38    update_equation 优化方法
39    '''
40    trainer = paddle.trainer.SGD(cost=cost,
41                                 parameters=parameters,
42                                 update_equation=momentum_optimizer)
43    return trainer

开始训练

有了训练器之前，再加上训练数据就可以进行训练了，我们还是使用我们比较熟悉的CIFAR10数据集，PaddlePaddle提供了下载接口，只要调用PaddlePaddle的数据接口就可以了。

合并模型

准备文件

合并模型是指把神经网络和训练好的模型参数合并生成一个可是直接使用的网络模型，合并模型需要两个文件：

  • 模型配置文件： 用于推断任务的模型配置文件，就是我们用了训练模型时使用到的神经网络，必须只包含inference网络，即不能包含训练网络中需要的label、loss以及evaluator层。我们的这里的模型配置文件就是之前定义的mobilenet.py的mobilenet神经网络的Python文件。

  • 参数文件： 使用训练时保存的模型参数，因为paddle.utils.merge_model合并模型时只读取.tar.gz，所以保存网络参数是要注意保存的格式。如果保存的格式为.tar，也没有关系，可以把里面的所有文件提取出来再压缩为.tar.gz的文件，压缩的时候要注意不需要为这些参数文件创建文件夹，直接压缩就可以，否则程序会找不到参数文件。保存参数文件程序如下：

1with open(model_path + '/model.tar.gz', 'w') as f:
2    trainer.save_parameter_to_tar(f)

预测图像

这个是我们的预测CPP程序，这个调用了PaddlePaddle的CAPI，通过这些接口来让模型做一个向前的计算，通过这个计算来获取到我们的预测结果。
因为PaddlePaddle读取的数据是float数组，而我们传过来的只是字节数组，所以我们要对数据进行转换，加了一个把字节数的jpixels的转成float数组的array。最后我们获得的结果也是一个float数组的array，这个是每个类别对于的概率：

 1JNIEXPORT jfloatArray
 2Java_com_yeyupiaoling_testpaddle_ImageRecognition_infer(JNIEnv *env,
 3                                                        jobject thiz,
 4                                                        jbyteArray jpixels) {
 5
 6    //网络的输入和输出被组织为paddle_arguments对象
 7    //在C-API中。在下面的评论中，“argument”具体指的是一个输入
 8    //PaddlePaddle C-API中的神经网络。
 9    paddle_arguments in_args = paddle_arguments_create_none();
10
11    //调用函数来创建一个参数。
12    CHECK(paddle_arguments_resize(in_args, 1));
13
14    //每个参数需要一个矩阵或一个ivector（整数向量，稀疏
15    //索引输入，通常用于NLP任务）来保存真实的输入数据。
16    //在下面的评论中，“matrix”具体指的是需要的对象
17    //参数来保存数据。这里我们为上面创建的矩阵创建
18    //储存测试样品的存量。
19    paddle_matrix mat = paddle_matrix_create(1, 3072, false);
20
21    paddle_real *array;
22    //获取指向第一行开始地址的指针
23    //创建矩阵。
24    CHECK(paddle_matrix_get_row(mat, 0, &array));
25
26    //获取字节数组转换成浮点数组
27    unsigned char *pixels =
28            (unsigned char *) env->GetByteArrayElements(jpixels, 0);
29    // RGB/RGBA -> RGB
30    size_t index = 0;
31    std::vector<float> means;
32    means.clear();
33    for (size_t i = 0; i < 3; ++i) {
34        means.push_back(0.0f);
35    }
36    for (size_t c = 0; c < 3; ++c) {
37        for (size_t h = 0; h < 32; ++h) {
38            for (size_t w = 0; w < 32; ++w) {
39                array[index] =
40                        static_cast<float>(
41                                pixels[(h * 32 + w) * 3 + c]) - means[c];
42                index++;
43            }
44        }
45    }
46    env->ReleaseByteArrayElements(jpixels, (jbyte *) pixels, 0);
47
48    //将矩阵分配给输入参数。
49    CHECK(paddle_arguments_set_value(in_args, 0, mat));
50
51    //创建输出参数。
52    paddle_arguments out_args = paddle_arguments_create_none();
53
54    //调用向前计算。
55    CHECK(paddle_gradient_machine_forward(gradient_machine_, in_args, out_args, false));
56
57    //创建矩阵来保存神经网络的向前结果。
58    paddle_matrix prob = paddle_matrix_create_none();
59    //访问输出参数的矩阵，预测结果存储在哪个。
60    CHECK(paddle_arguments_get_value(out_args, 0, prob));
61
62    uint64_t height;
63    uint64_t width;
64    //获取矩阵的大小
65    CHECK(paddle_matrix_get_shape(prob, &height, &width));
66    //获取预测结果矩阵
67    CHECK(paddle_matrix_get_row(prob, 0, &array));
68
69    jfloatArray result = env->NewFloatArray(height * width);
70    env->SetFloatArrayRegion(result, 0, height * width, array);
71
72    // 清空内存
73    CHECK(paddle_matrix_destroy(prob));
74    CHECK(paddle_arguments_destroy(out_args));
75    CHECK(paddle_matrix_destroy(mat));
76    CHECK(paddle_arguments_destroy(in_args));

79}

这里要注意，这个的infer方法不是我们的真正调用C++函数的方法，我们C++的预测函数传入的是一个字节数组：

所以我们要把获得的图像转换成字节数组，再去调用预测的C++接口：

 1public String infer(String img_path) {
 2	//把图像读取成一个Bitmap对象
 3	Bitmap bitmap = BitmapFactory.decodeFile(img_path);
 4	Bitmap mBitmap = bitmap.copy(Bitmap.Config.ARGB_8888, true);
 5	mBitmap.setWidth(32);
 6	mBitmap.setHeight(32);
 7	int width = mBitmap.getWidth();
 8	int height = mBitmap.getHeight();
 9	int channel = 3;
10	//把图像生成一个数组
11	byte[] pixels = getPixelsBGR(mBitmap);
12	// 获取预测结果
13	float[] result = infer(pixels, width, height, channel);
14	// 把概率最大的结果提取出来
15	float max = 0;
16	int number = 0;
17	for (int i = 0; i < result.length; i++) {
18		if (result[i] > max) {
19			max = result[i];
20			number = i;
21		}
22	}
23	String msg = "类别为：" + clasName[number] + "，可信度为：" + max;
24	Log.i("ImageRecognition", msg);
25	return msg;
26}

其中我们调用了一个getPixelsBGR()方法，这个CIFAR图片在训练时的通道顺序为B(蓝)、G(绿)、R(红)，而我们使用Bitmap读取图像的通道是RGB顺序的，所以我们还有转换一下它们的通道顺序，转换方法如下：

 1public byte[] getPixelsBGR(Bitmap bitmap) {
 2	// 计算我们的图像包含多少字节
 3	int bytes = bitmap.getByteCount();
 4
 5	ByteBuffer buffer = ByteBuffer.allocate(bytes);
 6	// 将字节数据移动到缓冲区
 7	bitmap.copyPixelsToBuffer(buffer);
 8
 9	// 获取包含数据的基础数组
10	byte[] temp = buffer.array();
11
12	byte[] pixels = new byte[(temp.length/4) * 3];
13	// 进行像素复制
14	for (int i = 0; i < temp.length/4; i++) {
15		pixels[i * 3] = temp[i * 4 + 2]; //B
16		pixels[i * 3 + 1] = temp[i * 4 + 1]; //G
17		pixels[i * 3 + 2] = temp[i * 4 ]; //R
18	}
19	return pixels;
20}
