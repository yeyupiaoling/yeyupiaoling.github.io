GitHub地址：https://github.com/yeyupiaoling/LearnPaddle2/tree/master/note14

前言

如果读者使用过百度等的一些图像识别的接口，比如百度的细粒度图像识别接口，应该了解这个过程，省略其他的安全方面的考虑。这个接口大体的流程是，我们把图像上传到百度的网站上，然后服务器把这些图像转换成功矢量数据，最后就是拿这些数据传给深度学习的预测接口，比如是PaddlePaddle的预测接口，获取到预测结果，返回给客户端。这个只是简单的流程，真实的复杂性远远不止这些，但是我们只需要了解这些，然后去搭建属于我们的图像识别接口。

了解Flask

安装flask很简单，只要一条命令就可以了：

1pip install flask

同时我们也使用到flask_cors，所以我们也要安装这个库

1pip install flask_cors

创建一个 paddle_server.py文件，然后编写一个简单的程序，了解一些如何使用这个Flask框架，首先导入所需的依赖库：

1import os
2import uuid
3import numpy as np
4import paddle.fluid as fluid
5from PIL import Image
6from flask import Flask, request
7from flask_cors import CORS
8from werkzeug.utils import secure_filename

编写一个 hello_world()函数，使用 @app.route('/')是指定访问的路径，该函数的返回值是一个字符串 Welcome to PaddlePaddle：

1# 根路径，返回一个字符串
2@app.route('/')
3def hello_world():
4    return 'Welcome to PaddlePaddle'

然后启动这个服务，如果是在Ubuntu的话，可能是需要在root下执行这个程序。

1if __name__ == '__main__':
2    # 启动服务，并指定端口号
3    app.run(port=80)

然后浏览器访问 http://127.0.0.1，返回之前写好的字符串：

1Welcome to PaddlePaddle

要预测图片，上传图片是首要的，所以我们来学习如何使用Flask来上传图片。

  • secure_filename是为了能够正常获取到上传文件的文件名
  • /upload指定该函数的访问地址
  • methods=['POST']指定该路径只能使用POST方法访问
  • f = request.files['img']读取表单名称为img的文件
  • f.save(img_path)在指定路径保存该文件
 1# 上传文件
 2@app.route('/upload', methods=['POST'])
 3def upload_file():
 4    f = request.files['img']
 5    # 设置保存路径
 6    save_father_path = 'images'
 7    img_path = os.path.join(save_father_path, str(uuid.uuid1()) + secure_filename(f.filename).split('.')[-1])
 8    if not os.path.exists(save_father_path):
 9        os.makedirs(save_father_path)
10    f.save(img_path)
11    return 'success, save path: ' + img_path

然后再次启动服务

1if __name__ == '__main__':
2    # 启动服务，并指定端口号
3    app.run(port=80)

然后再创建 index.html文件，编写一个表单，指定表单提交的路径 http://127.0.0.1/upload，并设置表单提交数据的格式 multipart/form-data，而且支持表单提交方式是POST。

 1<!DOCTYPE html>
 2<html lang="en">
 3<head>
 4    <meta charset="UTF-8">
 5    <title>预测图像</title>
 6</head>
 7<body>
 8<!--上传图片的表单-->
 9<form action="http://127.0.0.1/upload" enctype="multipart/form-data" method="post">
10    选择上传的图像：<input type="file" name="img"><br>
11    <input type="submit" value="上传">
12</form>
13</body>
14</html>

打开我们编写的网页，选择需要上传的图片，点击上传。成功后返回类似以下的字符串，图片会保存在 images目录下。

1success, save path: images/65d7661a-3892-11e9-a7b7-f44d30185f58jpg

预测服务

在 paddle_server.py中添加一个图片文件预处理函数，这个函数的参数是根据已经上传并保存到服务器上图片的路径，根据处理对图片进行预处理，并返回处理否的图片数据：

 1# 预处理图片
 2def load_image(file):
 3    img = Image.open(file)
 4    # 统一图像大小
 5    img = img.resize((224, 224), Image.ANTIALIAS)
 6    # 转换成numpy值
 7    img = np.array(img).astype(np.float32)
 8    # 转换成CHW
 9    img = img.transpose((2, 0, 1))
10    # 转换成BGR
11    img = img[(2, 1, 0), :, :] / 255.0
12    img = np.expand_dims(img, axis=0)
13    return img

以下就是PaddlePaddle代码，这次我们使用《PaddlePaddle从入门到炼丹》十一——自定义图像数据集识别训练保存的预测代码。

1# 创建执行器
2place = fluid.CPUPlace()
3exe = fluid.Executor(place)
4exe.run(fluid.default_startup_program())
5
6# 保存预测模型路径
7save_path = 'infer_model/'
8# 从模型中获取预测程序、输入数据名称列表、分类器
9[infer_program, feeded_var_names, target_var] = fluid.io.load_inference_model(dirname=save_path, executor=exe)

以下就是这个后端服务项目的核心代码，该函数主要的功能是接受上传的图片，对图片进行预测，最后返回预测的结果。预测的方式和之前的图片预测的方式一样的。

 1@app.route('/infer', methods=['POST'])
 2def infer():
 3    f = request.files['img']
 4
 5    # 保存图片
 6    save_father_path = 'images'
 7    img_path = os.path.join(save_father_path, str(uuid.uuid1()) + '.' + secure_filename(f.filename).split('.')[-1])
 8    if not os.path.exists(save_father_path):
 9        os.makedirs(save_father_path)
10    f.save(img_path)
11
12    # 开始预测图片
13    img = load_image(img_path)
14    result = exe.run(program=infer_program,
15                     feed={feeded_var_names[0]: img},
16                     fetch_list=target_var)
17
18    # 显示图片并输出结果最大的label
19    lab = np.argsort(result)[0][0][-1]
20
21    names = ['苹果', '哈密瓜', '胡萝卜', '樱桃', '黄瓜', '西瓜']
22
23    # 打印和返回预测结果
24    r = '{"label":%d, "name":"%s", "possibility":%f}' % (lab, names[lab], result[0][0][lab])
25    print(r)
26    return r

启动服务

1if __name__ == '__main__':
2    # 启动服务，并指定端口号
3    app.run(port=80)

在 index.html文件增加一个表单，这个是访问的地址就是上面服务中的预测函数的访问路径：

 1<!DOCTYPE html>
 2<html lang="en">
 3<head>
 4    <meta charset="UTF-8">
 5    <title>预测图像</title>
 6</head>
 7<body>
 8<!--调用服务器预测接口的表单-->
 9<form action="http://127.0.0.1/infer" enctype="multipart/form-data" method="post">
10    选择预测的图像：<input type="file" name="img"><br>
11    <input type="submit" value="预测">
12</form>
13</body>
14</html>

打开我们刚才编写的网页，选择需要预测的图片，点击预测。成功后返回类似以下的字符串，识别类别的标签，名字，还有该类别的概率。

1{"label":1, "name":"哈密瓜", "possibility":0.982786}
