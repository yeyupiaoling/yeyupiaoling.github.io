《我的PaddlePaddle学习之路》笔记十三——把PaddlePaddle部署到网站服务器上

2020-03-23
   
PaddlePaddle 人工智能 夜雨飘零 深度学习
    0 0

*本篇文章基于 PaddlePaddle 0.11.0、Python 2.7

前言

如果读者使用过百度等的一些图像识别的接口，比如百度的细粒度图像识别接口，应该了解这个过程，省略其他的安全方面的考虑。这个接口大体的流程是，我们把图像上传到百度的网站上，然后服务器把这些图像转换成功矢量数据，最后就是拿这些数据传给深度学习的预测接口，比如是PaddlePaddle的预测接口，获取到预测结果，返回给客户端。这个只是简单的流程，真实的复杂性远远不止这些，但是我们只需要了解这些，然后去搭建属于我们的图像识别接口。

环境

  1. 系统是：64位 Ubuntu 16.04
  2. 开发语言是：Python2.7
  3. web框架是：flask
  4. 预测接口是：图像识别

flask的熟悉

安装flask

安装flask很简单，只要一条命令就可以了：

1pip install flask

同时我们也使用到flask_cors，所以我们也要安装这个库

1pip install flask_cors

主要安装的是这两个库，如果还缺少哪些库，可以使用pip命令安装，*代表读者缺少的库：

1pip install *

测试flask框架

我们来编写一个简单的程序，来测试我们安装的框架，使用@app.route('/')是指定访问的路径：

 1from flask import Flask
 2
 3app = Flask(__name__)
 4
 5@app.route('/')
 6def hello_world():
 7    return 'Welcome to PaddlePaddle'
 8
 9if __name__ == '__main__':
10    app.run()

然后在浏览器上输入以下地址：

1http://127.0.0.1:5000

然后浏览器会返回之前写好的字符串：

1Welcome to PaddlePaddle

文件上传

我们来编写一个上传文件的程序，这个程序比上面复杂了一点点，我们要留意这些：
secure_filename是为了能够正常获取到上传文件的文件名
flask_cors可以实现跨越访问
methods=['POST']指定该路径只能使用POST方法访问
f = request.files['img']读取表单名称为img的文件
f.save(img_path)在指定路径保存该文件

 1from werkzeug.utils import secure_filename
 2from flask import Flask, request
 3from flask_cors import CORS
 4
 5app = Flask(__name__)
 6CORS(app)
 7
 8@app.route('/upload', methods=['POST'])
 9def upload_file():
10    f = request.files['img']
11    img_path = './data/' + secure_filename(f.filename)
12    print img_path
13    f.save(img_path)
14    return 'success'

然后我们编写一个HTML的网页index.html，方便我们测试这个接口：

 1<!DOCTYPE html>
 2<html lang="en">
 3<head>

 6</head>
 7<body>

 9    选择要预测的图像：<input type="file" name="img"><br>
10    <input type="submit" value="提交">
11</form>

13</html>

最后我们在浏览器上打开着网页，选择要上传的文件，点击提交，如果返回的是success，那代表我们已经上传成功了，我们可以去到保存的位置查看文件是否存在。

我们这次使用的是第二章的MNIST手写数字识别的例子，因为这个训练比较快，可以更快的获取到我们需要的预测模型，代码也是类似的，详细可以读到第二章的代码，只是添加了生成拓扑的功能

1# 保存预测拓扑图
2inference_topology = paddle.topology.Topology(layers=out)

同时把测试部分去掉了，这样训练起来速度会更快：

把PaddlePaddle部署到服务器

首先我们要创建一个队列，我们要在队列中使用PaddlePaddle进行预测

1app = Flask(__name__)
2CORS(app)
3# 创建主队列
4sendQ = Queue()

同样我们要编写一个上传文件的接口：

 1@app.route('/infer', methods=['POST'])
 2def infer():
 3    # 获取上传的图像
 4    f = request.files['img']
 5    img_path = './data/' + secure_filename(f.filename)
 6    print img_path
 7    # 保存上传的图像
 8    f.save(img_path)
 9    # 把读取上传图像转成矢量
10    data = []
11    data.append((load_image(img_path),))
12    # print '预测数据为：', data
13
14    # 创建子队列
15    recv_queue = Queue()
16    # 使用主队列发送数据和子队列
17    sendQ.put((data, recv_queue))
18    # 获取子队列的结果
19    success, resp = recv_queue.get()
20    if success:
21        # 如果成功返回预测结果
22        return successResp(resp)
23    else:
24        # 如果失败返回错误信息
25        return errorResp(resp)

对于上传文件和保存文件的介绍在上一部分已经讲，接下来就是把图像文件读取读取成矢量数据：

1data = []
2data.append((load_image(img_path),))

load_image()这函数在之前使用的是一样的

1def load_image(img_path):
2    im = Image.open(img_path).convert('L')
3    im = im.resize((28, 28), Image.ANTIALIAS)
4    im = np.array(im).astype(np.float32).flatten()
5    im = im / 255.0
6    return im

然后就是使用主队列发送图像的数据和子队列。使用子队列的作用是为了在PaddlePaddle的预测线程中把预测结果发送回来。

1# 创建子队列
2recv_queue = Queue()
3# 使用主队列发送数据和子队列
4sendQ.put((data, recv_queue))

下面就是我们的PaddlePaddle预测线程

  • 因为PaddlePaddle的初始化和加载模型只能执行一次，所以要放在循环的外面。
  • 在循环中，要从主队列中获取图像数据和子队列
  • 使用图像数据预测并获得结果
  • 使用recv_queue把预测结果返回
 1# 创建一个PaddlePaddle的预测线程
 2def worker():
 3    # 初始化PaddlePaddle
 4    paddle.init(use_gpu=False, trainer_count=2)
 5
 6    # 加载模型参数和预测的拓扑生成一个预测器
 7    with open('../models/param.tar', 'r') as param_f:
 8        params = paddle.parameters.Parameters.from_tar(param_f)
 9
10    # 获取分类器
11    out = convolutional_neural_network()
12
13    while True:
14        # 获取数据和子队列
15        data, recv_queue = sendQ.get()
16        try:
17            # 获取预测结果
18            result = paddle.infer(output_layer=out,
19                                  parameters=params,
20                                  input=data)
21
22            # 处理预测结果
23            lab = np.argsort(-result)
24            print lab
25            # 返回概率最大的值和其对应的概率值
26            result = '{"result":%d,"possibility":%f}' % (lab[0][0], result[0][(lab[0][0])])
27            print result
28            recv_queue.put((True, result))
29        except:
30            # 通过子队列发送异常信息
31            trace = traceback.format_exc()
32            print trace
33            recv_queue.put((False, trace))
34            continue

回到infer()函数中，刚才已经是把数据发送出去了，并有预测结果发送回来，我们这里就接收预测数据，并把预测结果返回给客户端。

1# 获取子队列的结果
2success, resp = recv_queue.get()
3if success:
4    # 如果成功返回预测结果
5    return successResp(resp)
6else:
7    # 如果失败返回错误信息
8    return errorResp(resp)

最后的两个函数是格式化返回的数据，生成的是一个json格式的数据。

1# 错误请求
2def errorResp(msg):
3    return jsonify(code=-1, message=msg)
4
5# 成功请求
6def successResp(data):
7    return jsonify(code=0, message="success", data=data)

最后就是启动我们的预测线程和服务了：

1if __name__ == '__main__':
2    t = threading.Thread(target=worker)
3    t.daemon = True
4    t.start()
5    # 已经把端口改成80
6    app.run(host='0.0.0.0', port=80, threaded=True)

同样在浏览器上打开刚才创建的HTML网页index.html，要注意的是提交的action改成http://127.0.0.1/infer，选择要预测的图像，点击提交，便可以获取预测结果

2  "code": 0, 
3  "data": "{\"result\":3,\"possibility\":1.000000}", 
4  "message": "success"
