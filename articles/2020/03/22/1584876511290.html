Android在ImageView上直接显示网络图片

2020-04-04
   
Android 夜雨飘零
    0 0

在原生的ImageView中，没有一个方法是可以直接显示网络的图片的，当我们经常需要显示网络图片时，每次都有一大堆的操作，这会很麻烦，今天就教大家在ImageView上轻松显示网络图片。

自定义ImageView方法

写一个类让它继承ImageView，并增加一个 setImageURL(path)方法

压缩

这是比较简单的从网络获取照片，直接在ImageView上显示，但是你有没有考虑过如果网络的图片很大，已经超出了手机屏幕的大小，如果还是加载原图的话无疑是浪费内存，还有可能造成内存溢出，所以我们有必要对网络的图片进行压缩，下面就开始讲网络图片的压缩。

首先获取ImageView要显示的宽度和高度

 1    /**
 2     * 获取ImageView实际的宽度
 3     * @return 返回ImageView实际的宽度
 4     */
 5    public int realImageViewWith() {
 6        DisplayMetrics displayMetrics = getContext().getResources().getDisplayMetrics();
 7        ViewGroup.LayoutParams layoutParams = getLayoutParams();
 8
 9        //如果ImageView设置了宽度就可以获取实在宽带
10        int width = getWidth();
11        if (width <= 0) {
12            //如果ImageView没有设置宽度，就获取父级容器的宽度
13            width = layoutParams.width;
14        }
15        if (width <= 0) {
16            //获取ImageView宽度的最大值
17            width = getMaxWidth();
18        }
19        if (width <= 0) {
20            //获取屏幕的宽度
21            width = displayMetrics.widthPixels;
22        }
23        Log.e("ImageView实际的宽度", String.valueOf(width));
24        return width;
25    }

然后是通过网络图片的大小计算要压缩的比率

 1    /**
 2     * 获得需要压缩的比率
 3     *
 4     * @param options 需要传入已经BitmapFactory.decodeStream(is, null, options);
 5     * @return 返回压缩的比率，最小为1
 6     */
 7    public int getInSampleSize(BitmapFactory.Options options) {
 8        int inSampleSize = 1;
 9        int realWith = realImageViewWith();
10        int realHeight = realImageViewHeight();
11
12        int outWidth = options.outWidth;
13        Log.e("网络图片实际的宽度", String.valueOf(outWidth));
14        int outHeight = options.outHeight;
15        Log.e("网络图片实际的高度", String.valueOf(outHeight));
16
17        //获取比率最大的那个
18        if (outWidth > realWith || outHeight > realHeight) {
19            int withRadio = Math.round(outWidth / realWith);
20            int heightRadio = Math.round(outHeight / realHeight);
21            inSampleSize = withRadio > heightRadio ? withRadio : heightRadio;
22        }
23        Log.e("压缩比率", String.valueOf(inSampleSize));
24        return inSampleSize;
25    }

获得压缩比率后，就可以进行压缩了

 1    /**
 2     * 根据输入流返回一个压缩的图片
 3     * @param input 图片的输入流
 4     * @return 压缩的图片
 5     */
 6    public Bitmap getCompressBitmap(InputStream input) {
 7        //因为InputStream要使用两次，但是使用一次就无效了，所以需要复制两个
 8        ByteArrayOutputStream baos = new ByteArrayOutputStream();
 9        try {
10            byte[] buffer = new byte[1024];
11            int len;
12            while ((len = input.read(buffer)) > -1 ) {
13                baos.write(buffer, 0, len);
14            }
15            baos.flush();
16        } catch (IOException e) {
17            e.printStackTrace();
18        }
19
20        //复制新的输入流
21        InputStream is = new ByteArrayInputStream(baos.toByteArray());
22        InputStream is2 = new ByteArrayInputStream(baos.toByteArray());
23
24        //只是获取网络图片的大小，并没有真正获取图片
25        BitmapFactory.Options options = new BitmapFactory.Options();
26        options.inJustDecodeBounds = true;
27        BitmapFactory.decodeStream(is, null, options);
28        //获取图片并进行压缩
29        options.inSampleSize = getInSampleSize(options);
30        options.inJustDecodeBounds = false;
31        return BitmapFactory.decodeStream(is2, null, options);
32    }

缓存

236        return BitmapFactory.decodeStream(is2, null, options);
237    }
238
239    /**
240     * 获得需要压缩的比率
241     *
242     * @param options 需要传入已经BitmapFactory.decodeStream(is, null, options);
243     * @return 返回压缩的比率，最小为1
244     */
245    public int getInSampleSize(BitmapFactory.Options options) {
246        int inSampleSize = 1;
247        int realWith = realImageViewWith();
248        int realHeight = realImageViewHeight();
249
250        int outWidth = options.outWidth;
251        Log.e("网络图片实际的宽度", String.valueOf(outWidth));
252        int outHeight = options.outHeight;
253        Log.e("网络图片实际的高度", String.valueOf(outHeight));
254
255        //获取比率最大的那个
256        if (outWidth > realWith || outHeight > realHeight) {
257            int withRadio = Math.round(outWidth / realWith);
258            int heightRadio = Math.round(outHeight / realHeight);
259            inSampleSize = withRadio > heightRadio ? withRadio : heightRadio;
260        }
261        Log.e("压缩比率", String.valueOf(inSampleSize));
262        return inSampleSize;
263    }
264
265
266    /**
267     * 获取ImageView实际的宽度
268     *
269     * @return 返回ImageView实际的宽度
270     */
271    public int realImageViewWith() {
272        DisplayMetrics displayMetrics = getContext().getResources().getDisplayMetrics();
273        ViewGroup.LayoutParams layoutParams = getLayoutParams();
274
275        //如果ImageView设置了宽度就可以获取实在宽带
276        int width = getWidth();
277        if (width <= 0) {
278            //如果ImageView没有设置宽度，就获取父级容器的宽度
279            width = layoutParams.width;
280        }
281        if (width <= 0) {
282            //获取ImageView宽度的最大值
283            width = getMaxWidth();
284        }
285        if (width <= 0) {
286            //获取屏幕的宽度
287            width = displayMetrics.widthPixels;
288        }
289        Log.e("ImageView实际的宽度", String.valueOf(width));
290        return width;
291    }
292
293    /**
294     * 获取ImageView实际的高度
295     *
296     * @return 返回ImageView实际的高度
297     */
298    public int realImageViewHeight() {
299        DisplayMetrics displayMetrics = getContext().getResources().getDisplayMetrics();
300        ViewGroup.LayoutParams layoutParams = getLayoutParams();
301
302        //如果ImageView设置了高度就可以获取实在宽度
303        int height = getHeight();
304        if (height <= 0) {
305            //如果ImageView没有设置高度，就获取父级容器的高度
306            height = layoutParams.height;
307        }
308        if (height <= 0) {
309            //获取ImageView高度的最大值
310            height = getMaxHeight();
311        }
312        if (height <= 0) {
313            //获取ImageView高度的最大值
314            height = displayMetrics.heightPixels;
315        }
316        Log.e("ImageView实际的高度", String.valueOf(height));
317        return height;
318    }

在这开源非常发达的时代，肯定会有大牛为我们做了个种各样的开源框架，根本不需要我们做这么复杂的工作，下面就简单使用图片加载框架Glide

在使用前要添加Glide的依赖库

1compile 'com.github.bumptech.glide:glide:4.0.0'

刚才的条件不变，把点击事件的操作换成下面两行代码

1String url = "https://pic.cnblogs.com/avatar/1142647/20170416093225.png";
2Glide.with(MainActivity.this).load(url).into(myImageView);
